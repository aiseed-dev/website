# 連載「Fable 5 が帰ってきた」

副題: 賢いチャットAIとして使うものではない ── その仕事は、Fable 5 に頼む価値があるか?

高コストな最上位モデル Claude Fable 5 を、「任せる価値のある仕事」にだけ
使うための見極め方を伝える連載。全6回+展望コラム(番外編)。

- **設計書(必読)**: [docs/plan/fable/plan.md](../../docs/plan/fable/plan.md)
  ── 連載構成・執筆ガイド・用語統一表・ファクトシート・制作フローのすべてが
  ここにある。各回の執筆は設計書に厳密に従うこと。
- **制作体制**: 設計 = Fable 5 / 執筆 = Sonnet 5 / 事実確認・最終判断 = 人間
- **公開 URL**: https://aiseed.dev/fable/

## フォルダ構成

```
articles/fable/
  01-what-is-fable-5/ja.md   第1回: Fable 5 とは何か、何が得意か
  02-…/ja.md                 第2回: コストの仕組みとモデルの使い分け
  …                          (設計書「3. 連載構成」の順)
```

- 各回は `NN-slug/ja.md`(日本語のみ。英語版を出す場合は同フォルダに `en.md`)。
- 章ラベルは「第N回」。番外編はフロントマターの `chapter_label: 番外編` で
  上書きする。
- `prev_slug` / `next_slug` で前後の回をつなぐ。

## ビルド

```
python3 tools/build_article.py articles/fable/01-what-is-fable-5/ja.md
python3 tools/build_article.py --all
```

## 執筆時の注意(設計書の要点)

- 数値・事実は設計書「5. ファクトシート」にあるものだけを使う。それ以外は
  本文に書かず【要確認: 〜】と印を付ける。
- ★印の項目(料金・利用上限・提供状況)は執筆のたびに一次情報で再確認。
- 記事末尾の注記は固定文言:
  「本記事の情報は◯年◯月時点のものです。料金・提供状況は変わる可能性が
  あるため、最新は Anthropic 公式情報をご確認ください。」
- 各回の末尾で「その仕事は、Fable 5 に頼む価値があるか?」に必ず答える。
