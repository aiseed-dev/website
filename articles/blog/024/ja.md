# Linux + OnlyOffice ── 情報処理を、自分の手に

「LinuxにしたいけどExcelが開けない」
「会社の文書を、よそのクラウドに預けるのは不安」

この記事は、その解き方をまとめます。結論から言えば、**Windows + Microsoft 365 は、次の組み合わせで置き換えられます**。

- **文書（Word・Excel・PowerPoint）** → OnlyOffice Desktop Editors
- **ファイル共有・社内外の協働** → OnlyOffice DocSpace Community
- **複雑なマクロ・VBA だけ** → Python

ただし主眼は「安いOfficeへの乗り換え」ではありません。**情報処理を、特定の会社の製品の中に閉じ込めず、自分（自分の組織）の手に保つ**こと──そのための道具立てです。

## なぜやるのか ── 安さではなく、違う価値

MicrosoftやGoogleをやめれば費用は浮きます。でも、本当の理由は別のところにあります。

- **文書とデータが、自分のものになる。** よそのクラウドに預けず、自分のマシン・自分のサーバーに置ける。
- **払っていたお金が、人に回る。** ライセンス料は、毎年消えて何も残りません。同じお金を、自社の人や能力に向ければ、年々積み上がる。**ライセンス料は何も産みません。人を育てます。**
- **一つの正解に頼らず、自分の解を持てる。** これが会社ごと・地域ごとに広がれば、解も、そこで育つ知識も多様になる。多様であること自体が、強さになります。

（Windowsを使うと、５年ごとの買い替えや、アップデートを強制されます。──しかい、Linuxだと静かです。これは副次的な利点です。）

## 道具立て

### 文書:OnlyOffice Desktop Editors

無料・オフラインで動き、MS Officeとの互換性が高く、UIもそっくり。`.docx` `.xlsx` `.pptx` をそのまま開け、Office経験者ならすぐ使えます。導入はFlatpakが手軽です。

```bash
flatpak install flathub org.onlyoffice.desktopeditors
```

### ファイル共有・協働:OnlyOffice DocSpace Community

ファイルを人と共有し、その場で協働する場——OneDrive / SharePoint の置き換えにあたる層には、**OnlyOffice DocSpace Community** を使います。文書共有とブラウザ共同編集を「room」（役割ごとにアクセス権を分けられる部屋）で構成し、同僚・取引先を room に招いて、ファイルを共有し、`.docx`等をその場で同時編集できます。編集エンジンはネイティブに統合済み。Community版はセルフホストで無料です。

公式スクリプトで入ります（root権限・Dockerが入ったLinuxで）。

```bash
curl -O https://download.onlyoffice.com/docspace/docspace-install.sh
bash docspace-install.sh docker        # Docker で入れる
# あるいは: bash docspace-install.sh package   # .deb / .rpm で入れる（Debianはこちらも素直）
```

終わったら、ブラウザでそのマシンのIPを開き、メール・パスワード・言語・タイムゾーンを設定するだけです。

> DocSpaceは組織の協働基盤なので、相応のサーバーが要ります（目安：8コア／16GB RAM／40GB＋swap 6GB）。組織にとっては妥当な一度きりの費用で、**人数ぶん毎年かかるMS 365の課金に比べれば、はるかに安く、しかも所有できます**。

置き場所も、自分で選べます。**社内の自分のマシン（手元のサーバー）に置けば、社内からは LAN で最速・追加費用ゼロ・データは完全に自分の手の中**。そして、外出先や社外の相手から繋ぐときは、**Cloudflare Tunnel** を使えば、**ポートを一切開けず・固定IPも要らず**に、安全に公開できます（無料。自宅・社内のルーターに穴を開けないので安全で、Cloudflare Access で「誰がログインできるか」も絞れます）。データは自分のマシン、外との境界はCloudflare——という切り分けです。

### 複雑なマクロ・VBA:Python

唯一そのまま移らないのが、ExcelのVBAマクロです（OnlyOfficeは独自のマクロ機能を持つため、そのままでは動きません）。でも、これは詰まりではなく、むしろ機会です。複雑な処理は、**Python** に外へ出す。AIに日本語で頼めばコードが返ってくるので、専門のプログラマでなくても移せます。表計算の中に閉じ込めず、言語で持つことで、後からいくらでも育てられます。

## 用途で道具を分ける

| 用途 | 道具 |
| --- | --- |
| 文書を書く（`.docx`等を開く・送る） | OnlyOffice Desktop Editors |
| ファイル共有・協働（同時編集・社外共有） | OnlyOffice DocSpace Community |
| 複雑なマクロ・VBA | Python |

一つの道具に全部やらせない。それぞれを得意な場所で使う。これだけで、Windows + Microsoft 365 の仕事は、置き換えられます。

---

## まとめ ── そして、拡張性が広がる

単なる置き換えて終わり、ではありません。**ここから、むしろ拡張性が広がります。**

Excelのマクロは表計算の中に閉じていましたが、Pythonに出せば、ほかのシステムとも・どんなデータとも繋げられる。OnlyOffice も DocSpace も、API やコネクタで、自分のアプリや業務に組み込める。**自分の手に持つほど、後から足せる・繋げられる幅が、広がっていきます。**

「Linuxにしたいけど踏み切れない」の答えは、OnlyOffice Desktop・DocSpace・Pythonで揃います。でも本当に手に入るのは、ソフトの置き換えではなく、**情報処理を自分の手に取り戻し、その先を自分で決めていく自由**です。
