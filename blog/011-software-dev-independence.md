# ソフトウェア開発の自立——Claude + Gemma 4 で作る4つのAIの役割分担

## Mythos時代の道具は揃った

2026年4月7日、AnthropicはProject Glasswingを発表した。Claude Mythosの脆弱性発見能力を、12社＋40超のインフラ組織に限定して提供する取り組みだ。Glasswingの参加企業の修正が一定段階に達すれば、Mythosは一般公開される。その時、修正が終わっていないシステムは全て標的になる。

しかし、これは恐怖の物語ではない。**自分で開発できる道具が、ちょうど同じタイミングで揃った**という話だ。

Anthropicのコーディングツール「Claude Code」は、リリース以来年間換算で10億ドルの収益を上げている。そしてAnthropicによると、このツールが新しいコードの70〜90%を生成している。Googleは Gemma 4 を Apache 2.0 のオープンソースで公開し、高性能なモデルをローカルで誰でも動かせるようになった。Gemini は Google検索インフラと統合され、deep research を1人でこなせる水準に達した。

**Claude + Gemma 4 + Gemini。この3つを役割で使い分け、本番環境からはAIを排除する。** これだけでGlasswingの外側でも構造的に安全な開発体制が作れる。

## 4つのAIの役割分担

AIを能力の高さで比較するのではない。**役割で分ける。**

### Claude——考える仕事（外部通信あり）

コーディング、レポート作成、構造分析、ブログ執筆。複雑な判断と創造が必要な作業。

Claude Code で開発する。設計をレビューする。ドキュメントを書く。記事の構造を組み立てる。

外部のAnthropicサーバーと通信する。その代わり、最先端の推論能力と Claude Code によるエージェント的な開発支援が手に入る。ここに出すのは「考えさせるべき題材」だけだ。生データや機密を丸ごと流し込むのではなく、構造・方針・コードレビューの対象として渡す。

### Gemma 4 CLI——処理する仕事（ローカル完結）

文書要約、データ集計、テキスト変換、定型分析。

外部通信ゼロ。自前サーバーで動く。Apache 2.0 で無料。
**エージェントとして常駐させない。CLIで呼び、一回で完結する。**

毎日の売上ログを要約する。CSVを整形する。議事録を箇条書きに直す。正規化・分類・テンプレート生成——定型的な情報処理を、ローカルの Gemma 4 に一発で任せる。常駐プロセスにしない。権限を持たせない。「入力 → 処理 → 出力」の一往復で終わらせる。これが攻撃面を生まない鍵だ。

### Gemini——調べる仕事（Google検索インフラ）

資料収集、決算分析、deep research。

Google は世界最大の検索・クローリング資産を持っている。Gemini はそれに直結している。「この業界の過去5年の決算推移」「この規制の各国比較」——こうした調査は、検索インフラと統合されたモデルが圧倒的に速い。

調べ物の出力は、人間が読む。そこから判断材料を抽出したら、Claude で分析し、必要に応じて Gemma 4 CLI で整形する。

### 本番環境——AIなし

静的HTML、Python実行コード、SQLite。**攻撃面ゼロ。**

公開サイトにAIエージェントを入れない。業務アプリにAIエージェントを常駐させない。本番環境で動いているのは、自分が読んで理解したコードだけ（＝Claude Code が生成し、人間がレビューしたもの）。外部APIに推論を投げない。LLMに判断を委ねない。

本番でAIを動かす誘惑は大きい。「チャットボットを置こう」「AIで要約を出そう」——しかし、それは Mythos クラスの攻撃者に入口を提供することだ。常時外部通信、システム権限、検証できない判断。構造的に攻撃面の拡大になる。

**AIは開発時に使う。本番では使わない。** この境界線を守れば、攻撃面はゼロに保てる。

## なぜこの4分割が機能するか

この役割分担は恣意的ではない。**それぞれのAIが得意な形で働かせ、苦手な所に置かない**という設計だ。

:::compare
| AI | 働かせ方 | データの流れ | 攻撃面 |
| --- | --- | --- | --- |
| Claude | エージェント的。対話と反復 | 考える題材だけ外に出す | 開発時のみ。本番になし |
| Gemma 4 CLI | ワンショット。入力→出力で終了 | ローカルから出ない | ゼロ（外部通信なし） |
| Gemini | 検索統合。調査クエリ | 公開情報中心 | 調査時のみ |
| 本番 | AIなし | 静的・ローカル完結 | ゼロ |
:::

:::chain
**4分割の原則：**
考える → Claude（高性能推論を借りる。題材だけ渡す）
処理する → Gemma 4 CLI（ローカルで完結。常駐させない）
調べる → Gemini（検索インフラに繋がったモデル）
動かす → AIなし（本番は静的／ローカル。攻撃面ゼロ）
**「どのAIを使うか」ではなく「どの仕事をどのAIに任せるか」で設計する**
:::

各AIの出力は、人間が受け取って次のAIに渡す。AI同士を自動でチェーンしない。AIエージェントが別のAIエージェントを呼び出す構造は、権限と通信経路が複雑化して、攻撃面を急速に広げる。**人間がハブになる。** これが4分割を安全に回す運用則だ。

## コードの7〜9割はAIが書いている

ここで見逃してはならない数字がある。

Claude Code は新しいコードの70〜90%を生成している。かつてソフトウェア開発は、大規模なチームと多額の予算を必要とした。SIerに数百万〜数千万円を払い、数ヶ月かけて作ってもらう世界だった。

今は違う。1人＋Claudeで、かつてのSIerチームと同等の開発ができる。ローカルの定型処理は Gemma 4 CLI が担う。調査は Gemini が加速する。

:::compare
| | SIerに依頼 | 1人＋Claude + Gemma 4 + Gemini |
| --- | --- | --- |
| 開発費 | 500万〜1,000万円 | Claudeの月額＋自前サーバー |
| 開発期間 | 2〜3ヶ月 | 24時間〜数日 |
| 構造の理解 | ブラックボックス | 自分で把握 |
| 定型処理 | 人月課金 or SaaS依存 | Gemma 4 CLI でローカル完結 |
| 調査 | 外注リサーチ | Gemini で即時 |
| 脆弱性対応 | SIerに再依頼 | 自分＋Claudeで即修正 |
| 攻撃面 | CMS＋DB＋API＋管理画面 | 静的HTML（ゼロ） |
:::

aiseed.devがまさにこの実証だ。30,000行以上のコード、42ページのHTML、26本の記事。全てClaude Codeが生成した。人間が書いたコードは0行。開発時間は約24時間。SIerに発注すれば500万〜1,000万円。

## AI軍拡競争——非対称の戦い

防御する側は倫理的制約の中でAIを使う。攻撃する側は制約なしにAIを使う。この非対称な戦いの中で、デジタルインフラは守られなければならない。

AnthropicはMythosを防御目的のみに限定した。しかし悪意を持つ攻撃者が開発した同様のAIシステムには、そのような制約はない。防御側がいかに強力なAIを持っていても、攻撃者の数が圧倒的に多ければ防ぎきれない。

だからこそ、**「守る」のではなく「攻撃面をなくす」ことが唯一の解だ。** そして4分割モデルは、まさにそのための設計だ。本番にAIを入れないことで、そもそもAI経由の侵入経路が存在しない状態を作る。

## 実践：開発の自立のための5原則

:::highlight
**原則1：AIは役割で分ける。本番には入れない。**
Claudeで考え、Gemma 4 CLI でローカル処理し、Gemini で調べる。
本番環境で動くのは静的HTMLとローカル実行コードだけ。AIエージェントを常駐させない。
:::

:::highlight
**原則2：CMSを使わない。**
公開サイトはMarkdown＋ビルドツール＋静的HTMLで十分だ。
Claude Code がビルドスクリプトを書いてくれる今、CMSに頼る理由はない。
:::

:::highlight
**原則3：データベースを本番に置かない。必要ならローカルでSQLite。**
SQLインジェクションの対象となるDBがなければ、攻撃は成立しない。
本番は静的配信。データ処理は Gemma 4 CLI と SQLite でローカル完結させる。
:::

:::highlight
**原則4：自分のサーバーを持つ。**
シンプルなLinuxマシン、Nginx、静的ファイル。Gemma 4 もこのサーバー上で CLI として動かす。
1サーバーに1アプリ。物理的・論理的に疎結合。
:::

:::highlight
**原則5：ブラックボックスを排除する。**
何をしているかわからないシステムは、守れない。
コードはAIに任せる。構造と目的を理解するのは人間の仕事だ。
Claude Code が書いたコードを人間が読み、構造を把握する。これが自立の核だ。
:::

## 自立は孤立ではない

開発の自立は、全てを独力でやることではない。**4つのAIが協力者だ。**

Claude Code がコードを書く。Claude が設計をレビューする。Gemma 4 CLI がローカルで日々の処理をこなす。Gemini が調査を加速する。人間がやることは、何を作るかを決めて、AIに指示を出して、結果を判断することだ。

:::chain
**自立の構造：**
人間 → 何を作るか決める → Claudeに指示する
Claude → コードを生成する → テストする → 修正する
Gemma 4 CLI → ローカルで定型処理をこなす
Gemini → 必要な情報を集めてくる
人間 → 結果を確認する → デプロイする（本番はAIなし）
**人間はコードを書かない。しかしシステムが何をしているかを理解し、全てをコントロールしている。**
:::

ソフトウェア開発がAIで完結するということは、**人間の時間とエネルギーを現実世界の仕事に集中できる**ということだ。

作物を育てる。製品を作る。人と向き合うサービスを運営する。これらはAIにはできない。土を触り、天候を読み、品質を判断し、顧客と対話する——現実世界の仕事には、人間の身体と判断が必要だ。

ソフトウェアはAIに任せればいい。必要なツールは24時間で作れる。定型処理はローカルで回る。調査は加速される。**だからこそ、人間は本当に価値のある仕事——現実世界のものづくり——に集中できる。**

これがソフトウェア開発の自立の本当の意味だ。ソフトウェアに縛られないこと。ソフトウェアを道具として使いこなし、自分の本業に戻ること。

## Mythosの公開に備える

Mythosが公開された瞬間、2つのことが同時に起きる。

1. 防御側が Mythos で自分のシステムをスキャンできるようになる
2. 攻撃側も Mythos で標的のシステムをスキャンし始める

修正までの時間差が生死を分ける。その時間差をゼロにできるかどうかは、今日の準備で決まる。

:::compare
| | 準備ができている組織 | 準備ができていない組織 |
| --- | --- | --- |
| ソースコード | 手元にある。全て把握 | SIerのブラックボックス |
| 構造の理解 | 自分で設計した | 誰も全体像を知らない |
| 定型処理 | Gemma 4 CLI でローカル完結 | 外部SaaSに依存 |
| 脆弱性の修正 | 自分＋Claudeで即修正 | SIerに依頼→数週間〜数ヶ月 |
| 攻撃面 | 静的HTML。スキャン対象がない | CMS＋DB＋API＋AIエージェント |
| Mythos公開後の生存 | **構造的に安全** | **最初の標的になる** |
:::

:::highlight
**今日から始められること：**
1. Linux を入れてターミナルに慣れる。全ての出発点だ
2. Claude Code で開発する練習を始める。AIと共同開発する能力を身につける
3. Gemma 4 を自前サーバーで CLI として動かせるようにする
4. Gemini で調査する習慣をつける（検索 + deep research）
5. ソースコードを手元に置く。GitHubに全てを公開する
6. 本番環境からAIエージェントを排除する（CMS→静的HTML、DB→ファイルベース）
**Mythosが公開されたとき、すぐにセキュリティチェックができる体制を作っておく。**
**そのためには、ソースコードがあること、構造が理解できていることが前提だ。**
:::

## 役割で分ければ、自立は始まる

Glasswing の12社は Mythos で自社システムを守る。その外側にいる無数の企業、自治体、個人は、自分で守るしかない。

しかし「自分で守る」は、「1人で戦う」ではない。**Claude + Gemma 4 + Gemini を役割で使い分け、本番にはAIを入れない。** この4つの配置こそが、Glasswingの外側で構造的に安全な開発体制だ。

**攻撃面をゼロにする。** これが最良の防御だ。Claude で考え、Gemma 4 CLI でローカル処理し、Gemini で調べる。本番は静的に保つ。

Mythos の公開は近い。準備の時間は半年から1年。道具は揃っている。あとは役割分担を決めて、今日始めるかどうかだ。

:::quote
考える仕事は Claude に。
処理する仕事は Gemma 4 CLI に。
調べる仕事は Gemini に。
動かす本番には AI を入れない。
4つの役割を分ければ、攻撃面はゼロになる。
Mythos が来ても、守る必要すらない構造が作れる。
道具は揃った。残っているのは決断だけだ。
:::
