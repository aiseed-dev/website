# 画面の見え方 (朝の確認用)

スクリーンショットの代わりに、各画面の構造と意図を文章で残す。
実機で立ち上げる前に、ここで全体像を把握できる。

## 全体構造

```
┌──────────────────────────────────────────────────┐
│ Debian 移行ウィザード                  aiseed.dev │  ← ヘッダー (固定)
├──────────────────────────────────────────────────┤
│  1. ようこそ › 2. アプリ棚卸し › ... › 10. まとめ │  ← Step indicator (10 step)
├──────────────────────────────────────────────────┤
│                                                  │
│  (現在の画面のコンテンツがここに入る)             │
│                                                  │
├──────────────────────────────────────────────────┤
│  [戻る]                       [Claude] [次へ]    │  ← ボトムバー (各ページ)
└──────────────────────────────────────────────────┘
```

ヘッダーと step indicator は `AppLayout` が常に表示。`ft.use_route_outlet()`
で各ページが中央のスロットに入る。step indicator の色は

- **太字 + アクセント色 (朱色)** = いま見ている step
- **緑** = 完了した step
- **薄いグレー** = まだ進んでいない step

step indicator は wrap=True なので、画面が狭い場合は折り返す。

**11 step の流れ:**

| # | path | フェーズ | 本書対応 |
|---|---|---|---|
| 1 | `/` | 入口 | — |
| 2 | `/inventory` | **計画 (pre-install)** ── アプリ棚卸し | 4 章 |
| 3 | `/replacements` | 同上 ── 代替候補 | 4 章 |
| 4 | `/hardware` | 同上 ── ハードウェア確認 | 6 章 |
| 5 | `/usb` | **作成** ── USB インストーラ | 7 章 |
| 6 | `/troubleshooting` | **post-install プレビュー** ── トラブル予防 | 8 章 |
| 7 | `/desktop` | 同上 ── デスクトップ環境 | 9 章 |
| 8 | `/ime` | 同上 ── 日本語入力 | 10 章 |
| 9 | `/install-plan` | 同上 ── アプリ導入計画 | 11 章 |
| 10 | `/operations` | **運用** ── dotfiles + メンテ | 12 + 17 章 |
| 11 | `/export` | **出力** ── Markdown レポート + Claude プロンプト | — |

## 各画面

### 1. ようこそ (`/`)

- 「このアプリでやること」のカード (4 つの bullet, 緑のチェックマーク付き)
- 「このアプリは何も書き換えません」というグレーの注意ボックス
- 右下に **[始める]** ボタン
- フッターに「底本: Claudeと一緒に学ぶDebian (全 24 章)」

意図: 初心者が「これは何のアプリか」「危なくないか」を 3 秒で判断できるように。

### 2. アプリ棚卸し (`/inventory`)

- 初回は中央に大きな検索アイコン + 「アプリを検出する」ボタン
- 検出中は ProgressRing 表示
- 検出完了で `DataTable` (アプリ名 / 提供元 / バージョン) が高さ 380 で
  スクロール表示
- 0 件のときは「自動検出できませんでした」のオレンジ INFO カード

意図: スキャンは数秒〜十数秒かかるので、進行表示が必須。テーブル形式は
普段 Excel を使っている層に親しい。

ボタン:
- **戻る** | **Claude 用プロンプトをコピー** | **代替候補を見る**

### 3. 代替候補 (`/replacements`)

- 凡例: 緑「そのまま置換可」/ 橙「要検討」/ 赤「代替なし」のチップ
- 検出アプリ × 代替テーブル のマッチ結果が `Card` のリストで出る
- 各カードに:
  - 太字でアプリ名 + 信頼度チップ
  - Debian 側の代替が tag-chips で並ぶ
  - 💡 で始まる注釈 (LibreOffice の互換性、VBA → Python、本書の章
    参照など)
  - **[OK にする]** / **[検討中]** トグル
- リスト全体は高さ 400 でスクロール

意図: ユーザーが「これは大丈夫」「これは要検討」を 1 クリックで仕分け
できる。OK にした項目は Markdown レポートにも反映される。

### 4. ハードウェア確認 (`/hardware`)

- ページ開いた瞬間に自動検出 (`use_state` で one-shot)
- 3 つのカード:
  1. **基本構成** ── OS / アーキ / CPU / メモリ / GPU を key-value で
  2. **ディスク** ── DataTable (デバイス / マウント / FS / 全体 / 空き)
  3. **注意点** ── 警告がなければ緑の OK バッジ、あれば橙の警告リスト

警告の例:
- Apple Silicon 検出 → 「Debian は Asahi Linux 経由なので推奨できない」
- NVIDIA GPU 検出 → 「プロプライエタリドライバの導入検討を」
- メモリ < 4GB → 「軽量 DE (XFCE/LXQt) の検討を」
- ディスク空き < 20GB → 「30GB 以上欲しい」

ボタン:
- **戻る** | **Claude 用プロンプトをコピー** | **再検出** | **USB 作成へ**

### 5. USB 作成 (`/usb`)

3 ステップカード:

**① Debian の ISO をダウンロード**
- ネットインストーラ (300〜500MB) を勧める
- 公式配布ページを開くリンク

**② 書き込み先の USB を選ぶ**
- 赤い ⚠ ボックス「選んだ USB の中身はすべて消えます」
- 検出された USB がカードで並ぶ (USB アイコン + 名前 + サイズ + ベンダー)
- 「これを使う」ボタンで選択。選択中は青色背景 + 「選択中」表示
- 「再スキャン」リンク

**③ USB に書き込む**
- OS 別に推奨 GUI ツール (Balena Etcher / Rufus) を表示
- Etcher のページを開くボタン
- ターミナル派のための `dd` コマンドを モノスペースで表示 +
  ▢ コピーボタン
- 真っ赤な ⚠ 「`of=` を間違えると本体ディスクが消える」警告

意図: 書き込みは v1 では自動化しない (危険なため)。コマンドと
ツールへの誘導まで。

### 6. 事前トラブル予防 (`/troubleshooting`) ── **第 8 章**

USB が用意できたあと、Debian 起動直後に出がちなトラブルを先に
頭に入れておくステップ。本書 第 8 章を wizard 化したもの。

3 つのカード:

**① 出がちな 7 カテゴリ**
- ディスプレイ / Wi-Fi / Bluetooth / サウンド / サスペンド /
  日本語入力 / 周辺機器
- 検出済みハードウェアから「当たりそう」を予測:
  - NVIDIA / AMD GPU → ディスプレイ ⚠
  - macOS から移行 → Wi-Fi / Bluetooth ⚠
  - すべての人 → サウンド・サスペンド・日本語入力 ⚠
- ⚠ が付いた行は背景が濃く + 橙のヒント文付き

**② Debian 起動後の診断コマンド**
- 5 件: `journalctl -b -p err` / `lspci -nnk` / `lsusb` /
  `dmesg | tail -50` / `systemctl --failed`
- 各コマンドに ▢ コピーボタン

**③ Claude にトラブルを聞くテンプレート**
- 本書 第 8 章にある統一プロンプト。下のボタンで一発コピー
- ハードウェア情報込みで生成 → ユーザーは症状 1 行と診断出力を
  埋めるだけ

ボタン: **戻る** | **Claude 用プロンプトをコピー** | **まとめへ**

意図: インストール前に「全部動くわけない、2-3 個は何かある」を
受け入れさせる。診断コマンドを「Debian 起動した瞬間にコピペ」
できるように用意しておく。

### 7. デスクトップ環境 (`/desktop`) ── **第 9 章**

GNOME / KDE Plasma / XFCE / Cinnamon / LXQt の 5 つから 1 つを選ぶ画面。

- 上部に「メモリ X GB の構成では Y がバランス良いです」のサマリー
  (検出済 RAM から推奨 DE を自動算出)
- 5 つの DE がカードで縦に並ぶ:
  - 名前 + (推奨なら「オススメ」バッジ) + (要件不足なら「⚠ メモリ XGB+ 推奨」橙バッジ) + 「これにする」ボタン
  - 一文サマリー
  - 「○ いいところ」(緑) と「△ 注意点」(橙) を 2 列で
- 選択中のカードは PRIMARY_CONTAINER 色で強調
- 下部に「後から入れる場合のコマンド」(`sudo apt install task-<de>-desktop`)
  をコピーボタン付きで提示

意図: 初心者は「どの DE を選ぶか」で迷うが、ハードウェアから現実的な
推奨を出すだけで決められるようにする。

### 8. 日本語入力 (`/ime`) ── **第 10 章**

Fcitx5 + Mozc の標準構成を導入するための画面。

3 つのカード:

**① 起動後に走らせる 4 手順** ── インストール / 既定設定 /
   ~/.profile への環境変数追記 / ログアウト再ログイン。
   各コマンドにコピーボタン。

**② よくある困りごと** ── 5 件 (半角/全角キー / 変換キー / 辞書登録 /
   Wayland / Electron アプリ) を Q&A 形式で。

**③ うまく動かないときの診断** ── 環境変数確認と `fcitx5-diagnose` を
   コピーボタン付きで。

### 9. アプリ導入計画 (`/install-plan`) ── **第 11 章**

ステップ 3 (代替候補) で「OK」を押した代替について、Debian での
インストール方法をまとめる画面。

- 上部に統計カード 4 枚 (対象アプリ / apt で入る / Flatpak / 手動)
- 中央に各アプリの詳細カード:
  - 「検出名 → 代替名」のタイトル
  - メソッドバッジ (青 apt / 紫 Flatpak / 橙 手動)
  - コピーボタン付きのコマンド
  - 💡 で始まる注釈
- 下部に **「一括インストールスクリプト」** ── apt と Flatpak と手動を
  分けてまとめた 1 枚スクリプト。「全部コピー」ボタンで Debian 起動後に
  ターミナルに貼って実行できる

データソース: `data/install_commands.py` に 100+ 件のアプリ →
Debian 導入方法のマップ。

### 10. 運用と長期メンテ (`/operations`) ── **第 12 章 + 第 17 章**

2 つのカード:

**① 設定ファイルを Git で管理 (第 12 章)**
- 管理対象のファイル / フォルダを 7 件、モノスペース + 説明の 2 列表
- 初期化スクリプト例 (`mkdir ~/dotfiles && git init && cp & symlink`)
  をコピーボタン付きで提示
- 「秘密鍵 / API トークンは絶対にコミットしない」の橙色注意

**② アップデートとメンテナンス (第 17 章)**
- 5 件のコマンドをコピーボタン付きで:
  - 週 1: `sudo apt update && upgrade -y`
  - 月 1: `sudo apt autoremove --purge && apt clean`
  - 週 1: `flatpak update -y`
  - ディスク容量確認
  - メジャー更新前: `timeshift --create`

### 11. まとめ (`/export`)

- 上部に 5 つの統計カード (検出アプリ数 / 代替確認 / 要検討 /
  ハードウェア警告 / 選択 USB)
- 「このあとの進め方」カード (4 つの番号付きステップ + 本書へのリンク)
- ボタン: **戻る** | **Markdown レポートを保存** |
  **Claude 用プロンプトをコピー**
- 保存に成功すると緑文字でパスが表示される (`~/debian-migrate-report-...md`)

Markdown レポートには:
- ハードウェア (警告含む)
- 代替検討の表 (検出名 / 候補 / 評価 / 選択 / 注)
- 検出アプリの全リスト
- 選択した USB
- 次のステップ

## キーボード / マウス操作

- 全ての主要ボタンは `FilledButton` / `OutlinedButton` で Material 3 標準
- ステップ間の遷移は明示的なボタンクリックのみ (誤遷移防止)
- データテーブルとスクロール領域はマウスホイール / トラックパッドで
  スクロール可能
- Markdown レポートのパスは `selectable=True` でコピー可

## 色とテーマ

- アクセント (Material 3 シード): `#c8442a` (朱色、aiseed.dev と統一)
- ライト/ダーク両対応 (`page.theme_mode = LIGHT` が既定)
- 警告: 橙 (`ORANGE`)、エラー / 致命: 赤 (`ERROR` / `ERROR_CONTAINER`)
- 成功: 緑 (`GREEN`)

## まだ無い機能 (v2 以降)

- 検出アプリのフィルタリング / 検索
- 代替候補ごとの「これを Linux で試す」ボタン (Flatpak / AppImage の
  事前ダウンロード)
- 「並行稼働期間」の計画立て (本書 第 6 章後半)
- 設定の永続化 (~/.config/debian-migrate/state.json)
- ダウンロードした ISO の SHA256 検証
- 多言語化 (EN 版)

## 実機で確認するときの順序

1. `cd apps/debian-migrate && uv sync`
2. `uv run flet run src/debian_migrate` (ネイティブウィンドウで起動)
   - 失敗するなら `uv run flet run src/debian_migrate --web` で
     ブラウザで開く
3. ようこそ画面でテーマの朱色が出ているか確認
4. アプリ棚卸し → 自分の PC のアプリが出るか
5. 代替候補 → 自分が日常で使っているソフトが拾われているか
6. ハードウェア → CPU/メモリ/ディスク/GPU の値が合っているか
7. USB → USB を差して再スキャン → 検出されるか
8. まとめ → Markdown 保存ボタン → ホームディレクトリに .md ができるか
9. 各画面の「Claude 用プロンプトをコピー」 → claude.ai に貼って
   ちゃんと相談できる形か
