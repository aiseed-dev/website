## ビルドシステムもAIが作った

サイトのビルドツール自体がAIによるソフトウェア開発の実例だ。

```
記事をMarkdownで書く
→ build_article.py がカスタムブロックを解析
→ markdown-it-py（CommonMark準拠）でHTMLに変換
→ Jinja2テンプレートに流し込む
→ SEOタグを自動付与
→ 全42ページを一括ビルド
```

最初はPython-Markdownとf-stringテンプレートで作った。
途中でJinja2 + markdown-it-pyに移行した。
移行もClaude自身が行った。

人間は「この例を活かせるはずです」とリポジトリのURLを渡しただけだ。

## AI需要予測は根本的に間違っている

こ���サイトの開発で明らかになったことがある。

**AIの本質的な価値は「開発を支援すること」であり、「24時間クラウドで推論を回し続けること」ではない。**

Claudeと対話してコードを書き、設定ファイルを作り、SQLを変換する——この作業に巨大なGPUクラスターは要らない。開発者のマシンとAPIアクセスがあれば十分だ。

「AIが普及する→クラウドGPUが大量に売れる→データセンター投資が爆発する」

この前提が過学習だ。AIが訓練データに過学習して汎化性能を失うように、投資家は「IT→サーバー」のパターンに過学習して「AI→GPU」と思い込んでいる。

**AIを「使う側」に必要なのはGPUではなく、知性だ。**
