伝統野菜・料理辞典
マルチAIリサーチシステム
プロジェクト計画書 v2.0
作成日: 2026-01-29
# 1. エグゼクティブサマリー
本プロジェクトは、Claude Agent SDKとGemini APIを組み合わせたマルチAIリサーチシステムにより、イタリア・日本の伝統野菜および伝統料理に関する包括的な辞典データベースを構築します。
野菜と料理を関連付けることで、「この野菜でどんな伝統料理が作れるか」「この料理にはどの品種が最適か」という実践的な知識基盤を提供します。
AI活用戦略: Claude MAX（メイン調査・構造化）+ Gemini API $300クーポン（大量バッチ処理・クロスチェック）のハイブリッド運用により、コスト効率と品質を両立します。
# 2. プロジェクトスコープ
## 2.1 対象領域
| 地域 | カテゴリ | 対象 | 目標数 |
|---|---|---|---|
| イタリア | 伝統野菜 | トマト、ナス、ズッキーニ、豆類、葉野菜 | 350品種 |
| イタリア | 伝統料理 | パスタ、リゾット、前菜、メイン、保存食 | 500レシピ |
| 日本 | 伝統野菜 | 京野菜、加賀野菜、江戸野菜、在来種 | 500品種 |
| 日本 | 伝統料理 | 郷土料理、精進料理、保存食、発酵食品 | 800レシピ |
| 合計 |  |  | 850品種 + 1,300レシピ |

## 2.2 野菜と料理の関連付け
各野菜エントリーには関連する伝統料理へのリンクを、各料理エントリーには使用する伝統野菜へのリンクを双方向で設定します。これにより、以下のユースケースに対応します。
- 野菜起点: 「賀茂なすを使った伝統料理を探す」
- 料理起点: 「カポナータに最適なナス品種を知る」
- 季節起点: 「夏に収穫できる野菜とその料理を一覧する」
- 地域起点: 「徳島の伝統野菜と郷土料理を探索する」
# 3. マルチAI活用戦略
## 3.1 AIモデル役割分担
| AI | プラン/予算 | 役割 | タスク例 |
|---|---|---|---|
| Claude MAX | 月額$100（定額） | 高精度調査・複雑な構造化 | 深堀り調査、文化背景分析、品質検証 |
| Gemini API | $300クーポン | 大量バッチ処理・初期収集 | 基本情報収集、リスト生成、翻訳 |

## 3.2 処理フロー
- Gemini（初期収集）: 野菜・料理の基本情報を大量バッチ処理で収集
- Claude（深堀り）: 文化的背景、栽培ノウハウ、調理の科学などを詳細調査
- Gemini（クロスチェック）: 複数ソースによる情報検証、矛盾検出
- Claude（最終構造化）: JSON形式への変換、関連付け、メタデータ付与

# 4. データ構造設計
## 4.1 野菜エントリースキーマ
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意識別子（例: IT-VEG-TOM-001） |
| names | object | 多言語名称（local, japanese, english, scientific） |
| origin | object | 原産地情報（country, region, history） |
| characteristics | object | 外観・味・食感・栄養素 |
| cultivation | object | 栽培方法（播種期、土壌、気候、病害虫） |
| related_recipes | array | 関連する伝統料理IDリスト |
| sources / metadata | object | 参照URL、収集日時、信頼度スコア |

## 4.2 料理エントリースキーマ（新規）
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意識別子（例: IT-RCP-PAS-001） |
| names | object | 多言語名称（local, japanese, english） |
| origin | object | 発祥地域、歴史的背景、伝承 |
| category | string | 料理カテゴリ（前菜、主菜、保存食など） |
| ingredients | array | 材料リスト（伝統野菜IDへのリンク含む） |
| traditional_method | object | 伝統的な調理法・手順・コツ |
| variations | array | 地域バリエーション、家庭ごとの違い |
| cultural_context | object | 祭事、季節、食べ方の作法 |
| related_vegetables | array | 使用する伝統野菜IDリスト |
| sources / metadata | object | 参照URL、収集日時、信頼度スコア |

# 6. 技術実装詳細
## 6.1 Claude Agent SDK構成
- query(): 単発の基本情報収集（野菜名→基本データ）
- ClaudeSDKClient: マルチターン深堀り（栽培法→病害虫→対処法の連鎖）
- WebSearch: 自律的Web検索（イタリア語・日本語サイト直接調査）
- Read: 画像分析（野菜の外観特徴、料理の盛り付け）
## 6.2 Gemini API構成
- Gemini 2.0 Flash: 高速バッチ処理（基本情報収集）
- Gemini 1.5 Pro: 長文コンテキスト処理（文献分析）
- Google Search統合: Grounding機能による検索連携
## 6.3 システムアーキテクチャ
FastAPIバックエンドから両AIを呼び分け、結果をJSON形式で統合保存します。Flutterフロントエンドは自己完結型Widgetで辞典閲覧・検索UIを構築します。
# 7. AIseed統合
- Grow連携: 栽培カレンダー、観察ガイド、収穫タイミング通知
- Learn連携: 伝統野菜・料理に関する学習コンテンツ自動生成
- 富山家藍栽培: 同システムを藍栽培・藍染料理の知識収集に応用
- BYOA対応: ユーザー自身のClaude Pro/Geminiでカスタム調査可能

# 9. 結論
Claude MAXとGemini APIのハイブリッド活用により、伝統野菜850品種と伝統料理1,300レシピを網羅した辞典データベースを、3ヶ月・低コストで構築できます。
