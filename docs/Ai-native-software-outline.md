# AIネイティブな仕事の作法 - ソフトウェア開発編 章立て案

## 第1章 AIがコードを書く能力で人間トップクラスに到達した
- 競技プログラミングでのAIの実績(Codeforces レーティング2700前後)
- 月額数千円で世界トップ層のコード生成能力にアクセスできる事実
- この事実から議論の出発点を設定する

## 第2章 保守フェーズの構造変化こそ本質
- コーディングコスト低下は氷山の一角にすぎない
- 保守対象が「コード」から「設計・仕様」へ移行する
- レガシーコードの解読コストが消滅する
- テストとドキュメントの自動同期
- ただし人間がアーキテクチャの主導権を握り続けることが必須条件
- 主導権を失えばAI生成による技術的負債が急速に蓄積する

## 第3章 コーダーの仕事はなくなる
- 「コーダー」の定義(コードを書くこと自体を仕事の中心に置く役割)
- 実行能力と判断能力の区別
- 実行はAIに置き換わる、判断は人間に残る

## 第4章 ビルダーという役割
- 何を作るかを決め、AIに作らせ、出力を評価し、構造を統合する人
- aiseed.devの実証(個人で30,000行・40ページを24時間で構築)
- コーダーとビルダーの構造的な違い

## 第5章 顧客がAIと協働して開発する時代
- 顧客自身が9割を作り、解けない部分だけ外部に依頼する構造
- AIができないことはSIerにもできない理由
- 自分で学習することのコストが下がった事実

## 第6章 SIer委託モデルの構造的不経済
- 外注プロセス自体の手間(要件定義・ベンダー選定・契約・管理・検収)
- 社内担当者の見えない工数コスト
- 同じ手間があればAIネイティブで開発できてしまう

## 第7章 価格競争力の桁違いの差
- SIer発注(数千万〜数億円)とAIネイティブ開発(月額数十万円)の比較
- 10倍〜100倍の価格差は競争ではなく市場破壊
- 日本市場は世界一価格差が大きい(=機会も大きい)

## 第8章 ロックイン問題
- SIer委託の三重ロックイン(独自フレームワーク、独自抽象層、人的依存)
- PalantirのFDEモデルを典型例として分析(FDEが顧客内部に張り付くこと自体がロックイン、独自Ontology、長期契約)
- AIネイティブ開発は標準的なコードを生成し、抽象層を最小化する
- 別のAI、別のビルダーが引き継げる構造

## 第9章 各社がビルダーを雇用する時代
- ITを外部化することは業務を外部化することと同じ
- ビルダーは判断を売る専門職(弁護士・医師との構造的類似)
- 一般社員ではなく専門職としての処遇が必要
- ホームページ開発を例にしたコスト比較

## 第10章 日本のSIer業界の転換と雇用流動性
- 多重下請け構造が転換を容易にする逆説
- 元請けは下請け契約解消で雇用調整なしに転換可能
- 下請けの優秀な人材は元請けや独立に流れる
- 雇用流動性という条件と、長期業務委託などの中間形態
- 流動性は時間とともに高まる方向にある

## 第11章 数年で完了する構造転換
- 変化の連鎖(AI能力→コーダー消滅→ビルダー需要→SIer縮小→下請け廃業)
- 5年前後で主要な変化が完了する見通し
- 不可逆な変化として進む
