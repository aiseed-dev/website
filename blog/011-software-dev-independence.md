# AI時代のデスクワークとシステム設計——Windows/Office・社内・公開の3領域を建て直す

## 2026年4月、道具が揃った

2026年4月、画期的な道具が2つ揃った。

- **Claude Mythos**（2026年4月、Anthropic が Glasswing 経由で限定公開）：高度な論理推論、抽象的思考、政策・戦略立案、そしてコーディング。
- **Gemma 4**（2026年4月、Google が Apache 2.0 で公開）：高性能なオープンソースモデルが、**手元のマシンで、外部通信ゼロで動く**。高速処理、定型業務、完全なプライバシー保護。無料。

Claude がロジックをコードにする。Gemma 4 が手元で定型処理をこなす。**この2つで、業務をコードに置き換えられる。**

## 緊急——Mythos公開までに終わらせる

### Glasswing——12社の内と外

2026年4月7日、AnthropicはProject Glasswingを発表した。Claude Mythosは12社＋40超のインフラ組織に限定される。**Glasswingの外側の組織は、自分で守るしかない。**

Mythosは永久に封印されない。Glasswingの12社の修正が進めば、一般公開される。その瞬間、**防御側と攻撃側が同時にMythosを使い始める。**

### Windows / Office は間に合わない

Windows と Office は数十年分のレガシーコードベースを抱え、さらに Copilot が OS と全 Office 製品に統合されている。Mythos でスキャンしても、修正すべき箇所が膨大すぎて公開までに終わらない。**Microsoft に自分のセキュリティを委ね続ければ、Mythos 公開後も長期間、未修正の攻撃面の上で仕事をすることになる。**

### Copilot 自動化は Mythos 時代に使えない

Microsoft は Excel/Word の操作負担を **Copilot エージェントによる自動化** で解決しようとしている。しかしこの方向は、Mythos 時代には構造的に成立しない。

- **プロンプトインジェクション**：文書に仕込まれた指示で Copilot が乗っ取られる。
- **データ主権の喪失**：社内文書が外部 AI に送信される。
- **思考の退化**：考える作業まで任せると、人間の判断力が育たない。

Copilot は「便利だが脆弱で検証できない自動化」を増やしている。Mythos が来れば全部入口になる。

### カウントダウン

:::chain
Glasswingの12社の修正 → Mythos一般公開
→ 攻撃側も同じ能力を手に入れる
→ Windows/Office レガシー、社内システム、公開サイト——全部スキャンされる
→ 修正が間に合わないシステムから順に陥落する
→ **準備の時間は半年〜1年。公開後では遅い。**
:::

## AIは役割で分ける

能力で比較しない。**役割で分ける。**

- **Claude Mythos**：考える仕事。コーディング、構造分析、戦略、執筆。渡すのは題材だけ。
- **Gemma 4 CLI**：処理する仕事。要約、集計、変換、定型分析。外部通信ゼロ。手元のマシンで動く。**常駐させない。CLIで一回完結。**
- **Gemini**：調べる仕事。資料収集、決算分析、deep research。
- **本番環境**：AIなし。静的HTML、Python、SQLite。**攻撃面ゼロ。**

:::chain
**4つの原則：**
考える → Claude Mythos
処理する → Gemma 4 CLI
調べる → Gemini
動かす → AI なし
**人間がハブ。AI同士を自動でチェーンしない。**
:::

## 3つの問題領域を建て直す

Glasswingの外側の組織にとって、守るべき領域は3つある。それぞれ問題の形が違うので、建て直し方も違う。

### 1. Windows / Office 問題（デスクトップ）

**問題：**

- バイナリファイル（Excel、Word、PowerPoint）。`git diff` で差分が取れない。
- 履歴が残らない。「誰が・いつ・なぜ変更したか」が頭の中にしかない。
- 手作業が中心。セルを埋める、書式を整える——判断していない操作に業務時間を消費する。
- 年間数百件の脆弱性。ファイルを開くだけ、プレビューするだけで攻撃が成立するゼロデイ。
- 密結合エコシステム（Windows ＋ Entra ID ＋ Azure ＋ SharePoint）。1箇所突破で全体に波及。
- Copilot 統合で攻撃面がさらに拡大。

**建て直し：**

- **Linux に移る。** デスクトップを Windows から離す。
- **バイナリ → テキスト：** Markdown、CSV、Python、JSON に移行。全部 Git に入れる（Doc as Code）。
- **Excel/Word 業務を観察 → ロジックを抜き出す → Claude にコード化を依頼。**
- **機密を含む定型処理は Gemma 4 CLI で一発処理。** 常駐させない。
- Copilot エージェントは使わない。

### 2. 社内システムの問題

**問題：**

- 旧来の業務システム、ERP、基幹業務、Access DB、オンプレアプリ。
- SIer が納品したブラックボックス。中身がわからない。
- 構造を説明できる人間が社内にいない。
- 脆弱性が報告されても、SIer に再依頼で数週間〜数ヶ月かかる。
- 外部 API 連携、VPN 越しアクセス、リモートデスクトップで外部からの経路が残る。
- AIエージェントを内部に入れる動きが「効率化」の名で拡大している。

**建て直し：**

- **SIer に任せない。自分＋Claude で建て直す。** 24時間で動くものができる。
- 業務フローを観察 → **人間が認めたロジックだけをコード化する**（Python、SQL、シェル）。
- **データは手元のマシンで SQLite。センターの DB は最小限に。**
- ソースコードは Git で管理する。構造は自分で把握する。
- **社内システムに AI エージェントを入れない。** Copilot も、Gemma 4 の常駐も、入れない。動くのは人間が認めたコードだけ。
- **Mythos が来ても、ソースが手元にあれば自分で修正できる。**

### 3. 公開システムの問題

**問題：**

- Webサイトに CMS（WordPress 等）。年間11,000件超の脆弱性。
- プラグインが 97%、認証不要で悪用可能が 57%。
- 公開 API、管理画面、データベース——攻撃面が広い。
- AIエージェント統合（チャットボット、AI検索）でさらに拡大。
- WAF で87.8%の攻撃を素通ししている。

**建て直し：**

- **CMS を捨てる。** Markdown ＋ ビルドツール ＋ 静的 HTML。
- **DB を本番に置かない。** 必要なら手元で SQLite。
- **本番に AI を入れない。** チャットボットも AI 検索も廃止。
- サーバーはシンプルに。**Linux ＋ Nginx ＋ 静的ファイル** だけ。
- 1サーバーに1アプリ。疎結合。
- **攻撃面をゼロにする。** スキャンしても何も見つからない状態を作る。

## 実践：6原則

:::highlight
**原則1：AI を使うのは、権限と責任を持つ人だけ。**
AI は責任を取れない。判断の結果に対して責任を取るのは人間だ。さらに AI は**それ自体が脆弱性の元**になる——外部通信、プロンプトインジェクション、検証できない判断。だから組織として「誰が AI を使うか」を決める。権限のある人が題材を選び、結果を判断し、責任を取る。**丸投げされた AI は、責任の所在を曖昧にし、攻撃面を広げる。**
:::

:::highlight
**原則2：AIは役割で分ける。本番には入れない。**
Claude Mythosで考え、Gemma 4 CLIで処理し、Geminiで調べる。本番は静的。
:::

:::highlight
**原則3：脱バイナリ。Doc as Code。**
Markdown、CSV、Python、JSON。全部 Git に入れる。
:::

:::highlight
**原則4：CMSとDBを本番に置かない。**
必要なら手元で SQLite。SQL インジェクションの対象がなければ攻撃は成立しない。
:::

:::highlight
**原則5：公開サイトは静的にする。**
Linux＋Nginx＋静的ファイルで十分。Gemma 4 は手元のマシンで CLI として動かす。
:::

:::highlight
**原則6：ブラックボックスを排除する。**
コードは AI に任せる。構造と目的を理解するのは人間の仕事。
「見えない自動化」を入れない。
:::

## 自立は孤立ではない

Claude Mythos がコードを書く。Gemma 4 CLI が手元で処理する。Gemini が調べる。**人間がやることは、何を作るか決めて、AI に指示して、結果を判断することだ。**

コードは書かなくていい。構造は理解しておく。

ソフトウェアを AI に任せれば、人間は現実世界の仕事に集中できる。作物を育てる。製品を作る。人と向き合う。**これが3領域を建て直した先の姿だ。**

## Mythos公開に備える——今日始める

公開までの時間は半年〜1年。**今日始めなければ、間に合わない。**

:::highlight
**今日から始めること：**
1. Linux を入れる。ターミナルに慣れる。
2. Claude Code で開発する練習をする。
3. Gemma 4 を手元のマシンで CLI として動かす。
4. Gemini で調査する習慣をつける。
5. Excel/Word 業務を観察して、定型処理をコードに置き換えていく。
6. 社内システムの再構築計画を立てる。SIer に頼らない選択肢を持つ。
7. 公開サイトを静的 HTML 化する。CMS と本番 DB を外す。
8. ソースコードを GitHub に置く。
9. 本番から AI エージェントを外す。
:::

## 役割で分ければ、自立は始まる

**Claude Mythos で考え、Gemma 4 CLI で処理し、Gemini で調べる。本番には AI を入れない。**

Windows/Office、社内システム、公開システム——3領域それぞれで攻撃面をゼロにする。**これが Mythos 時代に使える唯一の方法だ。**

:::quote
Windows/Office を離れる。
社内システムを自分＋Claudeで建て直す。
公開サイトを静的 HTML にする。
考える → Claude Mythos
処理する → Gemma 4 CLI
調べる → Gemini
動かす → AI なし
**Mythos 公開まで半年〜1年。今日始めろ。**
:::
