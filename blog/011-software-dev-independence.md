# AI時代のデスクワークとシステム設計——Claude Mythos と Gemma 4 で作る自立

## 2026年4月、道具が揃った

2026年4月7日、AnthropicはProject Glasswingを発表した。Claude Mythosは12社＋40超のインフラ組織に限定される。**Glasswingの外側の組織は、自分で守るしかない。**

だが、同じタイミングで画期的な道具が揃った。

- **Claude Mythos**：高度な論理推論、抽象的思考、政策・戦略立案。
- **Gemma 4**：高速処理、定型業務、そして**LGWAN（閉域網）やローカル環境での完全なプライバシー保護**（Apache 2.0、無料）。

機密を外に出さずに、AIの恩恵をフルに使える。**この2つがあれば、1人＋AIでSIerチーム相当の開発と業務ができる。**

## 緊急——Mythos公開までに終わらせる

**これは「いずれやればいい」話ではない。締切がある。**

Mythosは永久に封印されない。Glasswingの12社の修正が進めば、一般公開される。その瞬間、**防御側と攻撃側が同時にMythosを使い始める。**

:::chain
**残された時間：**
Glasswingの修正完了 → Mythos一般公開
→ 攻撃側も同じ能力を手に入れる
→ CMS、旧来のDB、AIエージェント本番常駐——全部スキャンされる
→ 修正が間に合わないシステムは最初の標的になる
→ **準備の時間は半年〜1年。公開後では遅い。**
:::

公開されてから動き出しても間に合わない。SIerに依頼すれば数週間〜数ヶ月かかる。**公開される前に、攻撃面をゼロにしておく必要がある。**

## AIは役割で分ける

能力の高さで比較しない。**役割で分ける。**

### Claude Mythos——考える仕事（外部通信あり）

コーディング、構造分析、戦略立案、執筆。複雑な判断と創造。Claude Code で開発する。渡すのは「考えさせるべき題材」だけ。生データや機密は渡さない。

### Gemma 4 CLI——処理する仕事（ローカル完結）

要約、集計、変換、定型分析。**外部通信ゼロ。LGWANや自前サーバーで動く。**

**エージェントとして常駐させない。CLIで呼び、一回で完結する。** 権限を持たせない。「入力→処理→出力」で終わり。これが攻撃面を作らない鍵だ。

### Gemini——調べる仕事

資料収集、決算分析、deep research。Google検索インフラに直結しているので速い。出力は人間が読む。

### 本番環境——AIなし

静的HTML、Python、SQLite。**攻撃面ゼロ。**

本番にAIエージェントを入れない。常時外部通信＋システム権限＋検証できない判断——それは Mythos クラスの攻撃者に入口を提供することだ。

**AIは開発で使う。本番には入れない。**

:::chain
**4つの原則：**
考える → Claude Mythos（題材だけ外に出す）
処理する → Gemma 4 CLI（ローカル／LGWAN、常駐させない）
調べる → Gemini（検索統合）
動かす → AIなし（本番は静的・ローカル）
**人間がハブ。AI同士を自動でチェーンしない。**
:::

## 脱バイナリ——ファイルをAIが読める形に

Word／Excelのバイナリファイルは、AI時代には非効率で不透明だ。

:::compare
| バイナリ中心 | テキスト中心 |
| --- | --- |
| Word / Excel / PowerPoint | Markdown / CSV / Python / JSON |
| 差分が見えない | Git で全履歴 |
| AIが扱いにくい | AIが直接読み書きできる |
| 手作業で整形 | 意図→AI→成果物（PDF等）を直接生成 |
:::

### Doc as Code

文書もコードと同じくGitで管理する。**誰が・いつ・なぜ変更したか**が全部残る。組織の記憶がブラックボックスから出てくる。

### 中間成果物の消滅

人間の「意図」が、AIを介して直接「成果物」に変わる。ファイルを手で整える時間がなくなる。

### Copilot統合への注意

Windows／OfficeのCopilotには、使う前に知っておくべき3つのリスクがある。

- **プロンプト・インジェクション**：文書に仕込まれた指示でAIが乗っ取られる。
- **データ主権の喪失**：文書が外部AIに送信される。
- **思考の退化**：考える作業まで任せると、判断力が育たない。

脱バイナリは「Microsoftを捨てる」話ではなく、**AIが読める形にデータを置き直す**話だ。結果としてCopilot依存からも自然に抜ける。

## システム設計——人間が認めたロジックをコードに

:::chain
**設計の基本：**
人間が認めたロジック → コード化 → 100%再現性で自動実行 → Gitに証跡
**コード化は Claude Mythos がやってくれる。難しくない。**
:::

- **AIエージェントに判断させない。** 本番では、人間が認めたロジックが動く。
- **自動化できないことは、人間が処理する。** その時にAIに助けてもらう。
- **責任は人間が取る。** AIはツール。判断と責任は人間の側にある。

## コードの7〜9割はAIが書いている

Claude Code は新しいコードの70〜90%を生成している。SIerに数百万〜数千万円払って数ヶ月待つ時代は終わった。

:::compare
| | SIer | 1人＋Claude Mythos＋Gemma 4 |
| --- | --- | --- |
| 費用 | 500万〜1,000万円 | Claudeの月額＋自前サーバー |
| 期間 | 2〜3ヶ月 | 24時間〜数日 |
| 構造 | ブラックボックス | 自分で把握 |
| 定型処理 | SaaS依存 | Gemma 4 CLI でローカル完結 |
| 脆弱性対応 | 再依頼で数週間〜数ヶ月 | 自分＋Claudeで即修正 |
| 攻撃面 | CMS＋DB＋API＋管理画面 | 静的HTML（ゼロ） |
:::

aiseed.devがその実証だ。30,000行のコード、42ページ、26記事。人間が書いたコードは0行。開発時間は約24時間。

## 実践：5原則

:::highlight
**原則1：AIは役割で分ける。本番には入れない。**
Claude Mythosで考え、Gemma 4 CLIで処理し、Geminiで調べる。本番は静的。
:::

:::highlight
**原則2：脱バイナリ。Doc as Code。**
Markdown、CSV、Python、JSON。全部Gitに入れる。
:::

:::highlight
**原則3：CMSとDBを本番に置かない。**
必要ならローカルでSQLite。SQLインジェクションの対象がなければ攻撃は成立しない。
:::

:::highlight
**原則4：自分のサーバーを持つ。**
Linux＋Nginx＋静的ファイル。Gemma 4 も同じサーバー上でCLIとして動かす。
:::

:::highlight
**原則5：ブラックボックスを排除する。**
コードはAIに任せる。構造と目的を理解するのは人間の仕事。
:::

## 自立は孤立ではない

Claude Mythos がコードを書く。Gemma 4 CLI がローカルで処理する。Gemini が調べる。**人間がやることは、何を作るか決めて、AIに指示して、結果を判断することだ。**

コードは書かなくていい。構造は理解しておく。

ソフトウェアをAIに任せれば、人間は現実世界の仕事に集中できる。作物を育てる。製品を作る。人と向き合う。**これがソフトウェア開発の自立の本当の意味だ。**

## Mythos公開に備える——今日始める

公開までの時間は半年〜1年。**今日始めなければ、間に合わない。**

:::highlight
**今日から始めること：**
1. Linux を入れる。ターミナルに慣れる。
2. Claude Code で開発する練習をする。
3. Gemma 4 を自前サーバーで CLI として動かす。
4. Gemini で調査する習慣をつける。
5. ソースコードをGitHubに置く。
6. 本番からAIエージェントを外す（CMS→静的HTML）。
:::

## 役割で分ければ、自立は始まる

**Claude Mythos で考え、Gemma 4 CLI でローカル処理し、Gemini で調べる。本番にはAIを入れない。**

これがGlasswingの外側で生き残る構造だ。攻撃面をゼロにすれば、守る必要もない。

**Mythos公開まで半年〜1年。今日始めろ。**
:::
