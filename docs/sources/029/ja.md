# 集中への警告と、手元の基盤

2026年6月、Microsoft の CEO サティア・ナデラが、AI業界に向けて警告を発した。6月14日のエッセイと、ウォール・ストリート・ジャーナルのインタビューでのことだ。それは技術の話ではなく、お金と力の話だった。

彼が言う問題の中心は、AIの「能力」ではなく「集中」だ。ごく少数の先端的なAIモデルが価値の大半を独占すれば、それぞれの産業の会社が、これまで自分たちを他社と差別化してきた固有の知識やノウハウを失ってしまう――そうナデラは警告する。

この警告は正しい。だから、その先を考える。

## 何が失われるのか

「固有の知識やノウハウを失う」とは、抽象的な話ではない。

製造業を例にとる。ある会社の差別化は、図面にある。NCプログラムにある。長年の不良対応で蓄積した品質の判断にある。取引先ごとに違う仕様の作り込みにある。これらは、どの教科書にも書かれていない。その会社が、現場で、時間をかけて、身体で覚えてきたものだ。

この固有の知識を、外部の少数のAIモデルに送って処理させるとどうなるか。データは、提供する事業者の管轄に入る。事業者が従う国の法の下に置かれる。中国には国家情報法があり、米国には CLOUD Act がある。Microsoft 365 も、OpenAI の ChatGPT も、Google も、データは事業者のクラウドへ渡る。そして、その知識を使って答えを出す能力は、自社ではなく、外のモデルの側に蓄積されていく。

差別化の源泉が、手元から、外へ移る。これがナデラの言う「失う」の中身だ。

## 集中は、資源を吸い上げる

集中の害は、知識の喪失だけではない。

少数の巨大なAIインフラは、現実の資源を一点に吸い上げる。データセンターは大量の電力を消費し、その需要が電気料金を押し上げる。AIサーバー向けの高帯域メモリと先端半導体に需要が集中し、メーカーは利益率の高いAI市場へ生産を振り向ける。その結果、一般向けのメモリと半導体の供給が逼迫し、価格が上がる。

これは、もう抽象論ではない。2026年6月25日、Apple が Mac と iPad を一斉に値上げした。Mac Mini は9万4800円から13万4800円へ。Apple 自身が値上げの理由をこう説明している――生成AI向けデータセンターの急拡大で、メモリやストレージの需要が異常な水準に達した、これほど急激な部品価格の上昇はかつてなかった、と。サムスンはデータセンター向けメモリを数カ月で大幅に値上げし、メモリ不足は2027年まで続くとの見方もある。Apple は一部の Mac で大容量メモリ構成そのものを削除した。

AIの集中は、AIを使わない人間にも、高い電気代と高いパソコンとして請求される。集中が続くほど、外にいる者が、その浪費を肩代わりさせられる。

## 送る、という一点

問題は、AIを使うことではない。送ることだ。

クラウド型のAIサービスは、データを送って初めて答えが返る。送れば、それは外に出る。出自がどこの国であれ、出れば、その事業者と、その国の制度の対象になる。狙われやすい産業ほど――製造業は、価値が高く守りが薄いという理由で、狙われやすい――この経路は重い。

逆に言えば、送らなければ、この経路は存在しない。

## 失わない形

失わない方法は、一つしかない。出さないことだ。データを手元に持つ。AIを手元で動かす。基盤を自分で持つ。

これは、特別な才能の話ではない。いま、それを成り立たせる部品は、すべて揃っている。

- **Caddy** ―― 外に面する唯一の窓口。HTTPS を自動で張るリバースプロキシ。
- **Forgejo** ―― 仕様もコードも、版管理された正本として置く。誰がいつ何を変えたかが残り、レビューを通して確定する。
- **PocketBase** ―― 認証。誰が、どの権限で、どのデータに触れてよいかを決める。Go の単一バイナリ。
- **OnlyOffice** ―― docx・xlsx・pptx を、データを構内に置いたまま編集する。日本語は BIZ UDPゴシック／BIZ UDP明朝 で、Windows と字組みがそろう。
- **Stalwart** ―― メール。SMTP・IMAP・スパム判定・暗号化を、Rust の単一バイナリ一個で賄う。固定IPと逆引きが整っていれば、自前で持てる。

これらを一台の Debian にまとめ、外に面する口は最小限に絞る。データは構内を出ない。

AIも同じだ。重みが公開され、手元のハードで動くモデルがある。書く側は、Cohere の **North Mini Code**（Apache 2.0、活性3BのMoE）。活性が3Bと軽いので、**Mac mini** のような机上の小さな一台でも、MLX や Ollama で実用速度が出る。大きなメモリは要らない。皮肉なことに、その Mac mini は今回のメモリ高騰で大容量構成を削られた。だが、手元で書かせるモデルに、大容量は要らない。

読む・見る側――同じ Cohere の **Command A+**（Apache 2.0、マルチモーダル、引用生成）。スキャンした図面まで読み、根拠を文書に紐づける――は規模が大きい。128GB の Ryzen AI Max+ 395 や 192GBクラスの機械、あるいはオンプレ配備で手元に置く。

AIには、いきなり本番を触らせない。書かせて、人が Forgejo 上で検証してから、適用する。版管理が、その検証の仕組みになる。設備（PLC等）を、AIに直接叩かせることはしない。

費用は、従量課金ではない。月額固定でもない。土台はオープンソースで、ほぼ実費で立つ。

## 二つの道

ナデラの警告を、もう一度置く。集中すれば、各産業の会社は、自分を差別化してきた固有の知識を失う。

この警告に対して、道は二つある。

一つは、便利さと引き換えに、知識を Microsoft や OpenAI のクラウドへ送り続ける道。もう一つは、Caddy と Forgejo と North Mini Code を手元に置き、知識を構内に留める道。

どちらを選ぶかは、それぞれの会社が決めることだ。ただ、部品はもう揃っている。手元に持つことは、もはや、できるかできないかの問題ではない。

そして、手元で動かすことには、もう一つの意味がある。少数のモデルを手元の機械で動かす選択は、それだけ、巨大データセンターへ電力と半導体を吸わせる構造に乗らない、ということだ。送らない者が増えるほど、集中への需要は細り、奪われていた資源は戻る。一社の自立は、その会社の知識を守るだけではない。外の皆が払わされている、電気代と半導体の請求書を、少しだけ軽くする。
