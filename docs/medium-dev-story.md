## The Build System — Also Built by AI

The site's build pipeline is itself a real-world example of AI software development:

```
Write articles in Markdown (with custom syntax extensions)
→ build_article.py parses custom blocks
→ markdown-it-py (CommonMark) converts to HTML
→ Jinja2 templates apply layout and SEO tags
→ sitemap.xml and robots.txt auto-generated
→ All 42 pages built in one pass
```

The system was initially built with Python-Markdown and f-string templates. Mid-development, Claude migrated it to Jinja2 + markdown-it-py. I just pointed Claude to a reference repo and said "this pattern should work."

## AI Demand Forecasts Are Fundamentally Wrong

Building this site made one thing clear:

**AI's essential value is in assisting development — not in running inference on the cloud around the clock.**

The premise behind the AI investment boom is: "AI adoption grows → cloud GPU demand explodes → data center investment skyrockets."

This premise is overfitting. Just as AI overfits to training data and loses generalization, investors have overfit to "IT grew → servers sold" and assume "AI grows → GPU servers sell."

Most companies don't need massive GPU clusters. They need Claude + a Linux machine. Configuration is faster locally than navigating the AWS console.

**What AI users need is not GPUs — it is intelligence.**
