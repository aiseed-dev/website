## はじめに——なぜ「教科書」か

Debian の入門記事はインターネット上に山のようにある。インストール手順、コマンド一覧、トラブル対処。それらは全て「すでに移行を決めた人」のための情報だ。

この教科書は、その手前にある問いを扱う。

**なぜ Debian なのか。なぜ今なのか。何が得られるのか。**

この三つの問いに答えられないまま手を動かしても、最初のトラブルで挫折する。逆に、この三つに自分の言葉で答えられれば、技術的な細部は後からついてくる。

教科書は六編で構成される。導入編で「なぜ」を問い、準備編で土台を固め、実践編で実際に移行する。応用編で日常を Debian に乗せ、組織編で複数人に展開し、最後に哲学編で長期的な意味を考える。

最初の一冊として、線を一本通して書いた。読み通したあとに「Debian を入れる」のではなく、「自分の暮らしを Debian という形で組み直す」ことを目的にしている。

## 第一章 なぜ Debian か——思想的な答え

### Linux ディストリビューションの中で Debian を選ぶ理由

Linux にはディストリビューションが何百もある。Ubuntu、Fedora、Arch、openSUSE、Mint、Manjaro。それぞれに支持者がいて、それぞれに長所がある。

ではなぜ Debian か。答えは思想にある。

Debian は世界で最も古いディストリビューションの一つであり、最初から「コミュニティが作る OS」だった。商業企業のマーケティング戦略でも、特定の創業者の所有物でもない。Debian Project には数千人のボランティアが参加し、技術的な意思決定は議論と投票で行われる。プロジェクト全体を縛るのは「Debian 社会契約（Debian Social Contract）」と「Debian フリーソフトウェアガイドライン（DFSG）」という二つの公開された約束だ。

この構造が何を意味するか。**Debian は誰かに買収されえない。** Red Hat は IBM に買収された。Ubuntu は Canonical という一企業の戦略に従う。SUSE はファンドの所有物だ。これらは悪いことではないが、企業の判断で方針が変わる可能性が常に残る。Debian は法人格を持たず、買収する対象がない。

長期的に依存できる相手は、買収できない相手だけだ。

### 「フリー」の意味

Debian の核は「フリーソフトウェア」という概念にある。フリーは無料という意味ではなく、「自由」という意味だ。具体的には四つの自由——使う自由、研究する自由、再配布する自由、改変して再配布する自由——を指す。

これがなぜ重要か。あなたが今使っているソフトウェアの多くは、ソースコードが公開されていない。中で何が起きているか分からない。バグがあっても直せない。サポートが終了したら使えなくなる。データはベンダーのサーバーに送られているかもしれない。

Debian は基本的に、ソースコードが公開されていて、誰でも検証でき、改変でき、自分の責任で使えるソフトウェアだけで構成されている。これは「監視に強い」「長持ちする」「自分の手で直せる」という三つの実用的な性質をもたらす。

### Mythos 時代の意味

[第5章「Mythosが来た」](/insights/mythos/)で書いたように、AI が数時間でゼロデイを発見できる時代が始まっている。守る側がコードを読めない OS を使い続けることは、攻撃面を黙って差し出すのと同じだ。

Debian は全コードが公開されている。脆弱性が見つかれば、コミュニティが直し、配布する。ベンダーの判断を待つ必要がない。Mythos 時代において、これは決定的な差になる。

## 第二章 なぜ今か——時代的な答え

### 三つの時計が同時に鳴っている

「いつか Linux に移ろう」と考え続けてきた人は多い。しかし「いつか」は過去三十年ずっと「いつか」のままだった。今、状況が変わった。三つの時計が同時に鳴っている。

**第一の時計：AI が Office と Windows のスキルを陳腐化させた。** Excel のマクロが書けることは、もはや競争力にならない。Claude や Gemma 4 に「この CSV を集計して」と指示すれば数秒で終わる。Office を使い続ける唯一の理由は「他の人が使っているから」だが、その理由は急速に弱まっている。

**第二の時計：Mythos 級 AI が、ブラックボックスの OS を維持不能にする。** Windows と Office は数十年分のレガシーコードを抱えており、Mythos 公開後の半年〜1年で攻撃面が露呈する。修正は Microsoft の判断と速度に依存する。一方、Debian は脆弱性発見から修正まで数日〜数週間で配信される。

**第三の時計：化石資源の制約が、ハードウェアの更新サイクルを変える。** 新しい Windows は新しい PC を要求する。古い PC は OS のサポート終了で電子ゴミになる。Debian は十年前の PC でも快適に動く。これは資源制約の時代に効いてくる差だ。

三つの時計のうち一つでも鳴れば移行を考える理由になる。三つ同時に鳴っているのが今だ。

### 「待つコスト」が「動くコスト」を超えた

二十年前、Linux への移行コストは大きかった。日本語入力、プリンタ、無線 LAN、ハードウェアサポート——どれも手間がかかった。今は違う。Debian 12 は USB に書き込んで起動するだけで、日本語環境も Wi-Fi も自動で動く。アプリケーションの大半はブラウザで完結する。

逆に、Windows を使い続けるコストが上がっている。アップデートで強制再起動、Recall や Copilot の押し付け、ライセンスとアカウント要件の変動。OneDrive にデータが勝手に上がる。広告がスタートメニューに出る。「自分の PC を自分が支配している」感覚が、年々薄れている。

天秤は傾いた。今、待つほうがコストが高い。

### 「個人＋AI」の時代との整合

[第9章「AIと個人事業」](/insights/ai-and-individual/)で書いた構造——一人＋AI で企業に匹敵する仕事ができる時代——は、Linux と相性がいい。個人で組み上げる環境は、ベンダーロックインを抱えていてはいけない。クラウド税、SIer 税、Office 税を払う前提では、個人の経済性が成り立たない。

Debian は「税」を引いた基盤として完璧に機能する。AI と組み合わせるなら、土台はオープンであるほどいい。

## 第三章 何が得られるか——実用的な答え

### 五つの具体的な獲得

抽象論を離れて、Debian に移行すると何が手に入るのかを五つ挙げる。

**一、所有感。** 自分の PC で何が動いているかを知ることができる。プロセスを見て、設定ファイルを読んで、必要なら直せる。これは技術力の問題ではなく、関係の問題だ。「借り物の OS」と「自分の OS」では、日々の使い方が変わる。

**二、速度。** 起動が速い。動作が軽い。古いハードウェアが蘇る。Windows 11 が要求するハードウェアの半分のスペックで、より快適な環境が手に入る。

**三、安定性。** Debian Stable は数年間ほぼ変わらない。ある日突然 UI が変わったり、機能が消えたりしない。アップデートは粒が細かく、巨大な一括更新で半日待たされることがない。

**四、コストのゼロ化。** OS、Office 相当、画像編集、動画編集、開発環境——日常業務に必要なソフトウェアの大半が無料で手に入る。年間数万円〜数十万円のソフトウェア支出が消える。

**五、長期的な自立。** 自分の暮らしを支える基盤を、自分でコントロールできる。これは、ベンダーの方針変更、価格改定、サブスクリプション化、サービス終了といった外部の波から距離を取れるということだ。

### 失うもの——正直に書く

獲得だけ書いて失うものを書かないのはフェアでない。

ゲームの一部は動かない。業界専用のソフトウェア（特定の会計、CAD、業務系）は代替が必要だ。同僚と Office ファイルを完全互換でやり取りする運用は、ひと工夫いる。Adobe の最新機能を使っている人は移行できない。これらは事実として認める。

しかし、これらの「失うもの」は、移行を諦める理由にはならない。準備編で、何を失い何を残すかを整理する方法を扱う。

### この教科書が約束すること

この教科書を読み通したとき、あなたは三つのものを得る。

一つ目は、**判断軸**だ。Debian に移るべきか、移らないべきか、いつ移るべきか、どこから移るべきか。これらに自分の言葉で答えられるようになる。

二つ目は、**手順**だ。実際にインストールし、設定し、日常業務を移し、開発環境を組み立てる具体的な道筋が分かる。

三つ目は、**思想**だ。なぜこの選択が、Mythos 時代と化石資源制約の時代において合理的なのか、その背景にある哲学が分かる。

導入編はここで終わる。次は準備編——実際に手を動かす前に、何を確かめておくかを整理する。
