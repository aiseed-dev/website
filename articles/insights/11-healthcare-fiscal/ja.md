## 軍需産業の世代交代——大型兵器からドローン+AIへ

このシリーズで見てきた構造を、産業の未来に当てはめる。
最初に再編されるのは軍需産業だ。

### 旧来の軍需産業——化石資源と高額単価のモデル

:::highlight
**旧来の軍需産業の化石資源依存：**
ジェット戦闘機 → ジェット燃料（灯油留分）＋複合材料（石油化学）
戦車 → 軽油＋合成ゴム＋高機能プラスチック
軍艦 → 重油＋塗料（石油系）＋配線被覆（PVC）
ミサイル → 固体推進剤の一部は石油化学由来＋電子部品（フォトレジスト）
兵站（ロジスティクス） → 軽油で動くトラックと輸送船が全て
:::

第10章で見たように、ドローン+AIが従来の大型兵器を無力化しつつある。
そして、大型兵器は化石資源なしには動かない。

:::chain
**旧来軍需産業の縮小構造：**
ドローン+AIが大型兵器を無力化 → 戦闘機・戦車の需要減少
化石資源の価格上昇 → 大型兵器の運用コスト上昇
化石資源の枯渇 → 大型兵器の製造自体が困難に
→ ロッキード・マーティン、レイセオン、BAEシステムズ——
→ **旧来型軍需産業の市場が構造的に縮小する**
:::

### しかし、これは軍需産業の消滅ではない

旧来の軍需産業の縮小と並行して、**ドローン+AI を中核に据えた
新しい軍需産業** が急速に台頭している。

第10章で見たように、ウクライナの **Brave1** エコシステムは、

- AI 関連プロジェクト登録数:**300超**
- 戦場で運用中の AI・コンピュータビジョン・システム:**70超**
- 機械視覚ドローンを開発する企業:**100超**
- 2026 年ドローン年産目標:**700万機**
- AI 自律航法モジュール:**1機あたり約 50 ドル**

を擁する、新しい軍需産業の中核を実戦のなかで作り上げている。

:::compare
| 旧来の軍需産業 | 新しい軍需産業 |
| --- | --- |
| ロッキード・マーティン(F-35、ミサイル) | **Anduril**(自律ドローン、AI 防衛 SaaS) |
| BAE Systems、Northrop Grumman | **Helsing**(欧州 AI 防衛、独・英・仏) |
| レイセオン(Patriot、ミサイル防衛) | **Quantum-Systems、Skydio、Shield AI** |
| 日本の重工業(MHI、川崎重工、IHI) | **Saker、ウクライナ Brave1 100+ 社** |
| 高額な単一プラットフォーム(数十億円〜) | 安価な分散ドローン(数万円〜数百万円) |
| 数十年の開発サイクル | 戦場フィードバックで数週間ごとに改良 |
:::

### 「軍需産業を持てる国」の境界が書き換わる

旧来の軍需産業は、数兆円の戦闘機開発予算を持つ大国だけのプレイヤーだった。
新しい軍需産業は、**$50 のAIモジュールと数万ドルのドローンが基本単位**
なので、中堅国でも自国産業として保有できる。

:::highlight
**産業構造の反転：**
旧来:大国だけが軍需産業を持てる → 中堅国は買うしかない
新しい:中堅国でも軍需産業を持てる → ウクライナが他国に輸出する側に回る
これが「安全保障の供与者」への転換の産業基盤である(第10章)。
:::

軍需産業で働く人の総数は減らないかもしれない。しかし **どこの国の、
どんな企業に勤めるか** が大きく書き換わる。
旧来型の防衛大手の雇用は減り、AI 防衛スタートアップ・ドローンメーカー・
戦場 SaaS 企業の雇用が増える。これは「衰退」ではなく **世代交代** である。

## IT産業の世代交代——既存クラウド・SaaSの衰退、AI企業の台頭、その先にOSS AI

次に再編されるのは IT 産業だ。これも軍需産業と同じく、**衰退ではなく世代交代** である。
ただし、**二段階の世代交代** という点に特徴がある。

### 第一の交代——既存IT企業の衰退

第6章、第7章、第8章の結論がここに集約される。

:::chain
**既存IT企業の縮小構造：**
AGIが採算に合わない（第6章）→ 過剰なAI投資の縮小
→ NVIDIAのGPU特需が剥落（第7章）
→ データセンター建設の減速
→ クラウド事業の成長鈍化（AWS、Azure、GCP）
CUDAロックインの崩壊（第7章）→ GPU市場が価格競争に
Oracle/Microsoft税の崩壊（第8章）→ ライセンス収入の減少
SIer税の崩壊（第8章）→ システムインテグレーター業界の縮小
:::

:::highlight
**縮小する側：**
NVIDIA → GPU需要のピークアウト
Oracle → PostgreSQL移行でライセンス収入消滅
Microsoft → Office/Azure/Windows税の段階的崩壊
Salesforce → Claude＋自前CRMへの移行
Accenture、NTTデータ等のSIer → Claudeが工数の大半を代替
大手コンサルティングファーム → Claudeが構造分析を代替
:::

### 第二の交代——AIネイティブ企業の台頭

旧来のIT企業が縮小する一方で、**最初から AI を中核に据えた設計**
の新しい企業群が急速に台頭している。

:::compare
| 旧来のIT企業 | 新しいAIネイティブ企業 |
| --- | --- |
| Microsoft(Office、Azure) | **Anthropic(Claude)、OpenAI(ChatGPT)** |
| Atlassian、ServiceNow、Salesforce | **Cursor**(AI ネイティブ IDE) |
| AWS の従来型ホスティング | **Vercel**(AI SDK、Edge AI、v0) |
| Oracle、SAP | **Hugging Face、Replicate**(モデルホスティング) |
| Adobe Creative Cloud | **Midjourney、Runway、ElevenLabs** |
| Accenture、IBM、NTTデータ | **Sourcegraph、Linear、Notion**(AI 中核に再設計) |
| Bloomberg、Reuters | **Perplexity、Glean**(AI 検索・調査) |
:::

これらは「既存製品にAI機能を後付け」したのではない。
**最初の行から、AI を前提に書かれたソフトウェア** である。

:::highlight
**新しい労働装備率：**
ベテラン SIer の数百人より、Cursor + Claude を使う 1 人の方が、
速く・正確にコードを書ける。
これは個別の生産性の話ではなく、**ソフトウェア産業の労働装備率
そのものが書き換わる** 構造である。
1人 + AI が、20人の旧来型開発チームを置き換える。
:::

### 第三の交代——OSS AIの広がり

ただし、AI ネイティブ企業の台頭は **過渡期の現象** である可能性が高い。
プロプライエタリな大規模言語モデルへの依存は、長期的には **OSS
(オープンソース) AI** に置き換わっていく。

:::chain
**OSS AIへの構造的シフト：**
Meta の Llama → 既に商用 AI と肩を並べる性能を OSS で公開
Alibaba の Qwen、DeepSeek、Mistral、Gemma → 商用版と同等性能を OSS で
量子化・蒸留技術 → ノートPC・スマホで70B級モデルが動く
ローカル推論コスト → API 利用料の 1/10 以下
→ プロプライエタリAIの価格的優位が崩れる
→ 企業・個人が自前でモデルをホスト
→ OSS AIが標準化する
:::

これは Linux と Windows の関係に近い。

:::highlight
**Linux/Windows パターンの再現：**
1990年代:商用 OS(Windows、Solaris、AIX)が支配的
2000年代:Linux がサーバーサイドで標準化
2020年代:クラウド・コンテナ・スマホ(Android)も Linux ベース
2025年:商用 LLM が支配的
2030年代:OSS AI がエッジ・組込み・サーバーで標準化
2040年代:商用 LLM は特定の高度用途に限定される
:::

20 年スパンで見れば、Anthropic や OpenAI は
**OSS への過渡期にイノベーションを牽引した存在** として振り返られる
可能性が高い。Cursor も Vercel も、その上で動く道具として残る。

その結果、

- **AI そのものは「インフラ」になる** ── 電気のように、誰でも安価に使える
- **AI で何をするか** が差別化要因になる ── 特定領域の知識・データ・関係性
- **個人と小組織が AI を所有する** ── 大企業の独占が解ける

第9章「AIと個人事業」で論じた「1人 + AI」の事業モデルが、
20 年スパンで主流になる構造的根拠は、ここにある。

> 旧来 IT の衰退、AI ネイティブ企業の台頭、その先に OSS AI が広がる。
> **一段階で終わる転換ではない、三段階の世代交代** である。
> どの段階で参入するかで、得られる成果が大きく違う。

## 受託型デスクワークの構造変化 ── 執行は AI、検証・修正・適応は人間

軍需産業と IT 産業の世代交代は、企業や国家のレベルの話だ。
しかし、より広範な影響が企業の内部で起きる ── **受託型デスクワーク
の構造変化**。

ここで誤解を避けたい。AI で消えるのは **デスクワーク全体ではない**。
書類作成・データ入力・レポート・契約レビュー・コーディング ──
どの業務も、

- **何を作るか・なぜ作るか** を決める **指示・要件設計**
- AI の出力を **検証・修正し、環境変化に適応させる** 継続的作業
- **出来上がったものを採用するかどうか** を決める **最終判断**

の三つは、**人間に残る**。AI はその間にある **純粋な執行(execution)**
の部分を高速に代替する。消えるのは「指示を受けて、変化のない環境で、
単純な執行を繰り返すこと」を専業とする労働である。

:::compare
| デスクワーク | 指示・要件設計 | 検証・修正・環境適応 | 純粋な執行(定型) |
| --- | --- | --- | --- |
| 書類作成 | 残る(何を作るか) | **残る**(出力の精度確認・体裁修正) | AI が省力化 |
| データ入力 | 残る(何を入力するか) | **残る**(妥当性検証・異常検知) | AI が省力化 |
| レポート作成 | 残る(問いの設定) | **残る**(数字の検証・解釈の修正) | AI が省力化 |
| 経理・会計 | 残る(取引の入力) | **残る**(税制変更への対応・承認判断) | AI が省力化 |
| 法務チェック | 残る(リスク観点の指定) | **残る**(規制変更への対応・契約判断) | AI が省力化 |
| 翻訳 | 残る(用途・トーン指定) | **残る**(業界用語・文脈の調整) | AI が省力化 |
| カスタマーサポート | 残る(エスカレーション基準) | **残る**(顧客状況の理解・例外対応) | AI が省力化 |
| プログラミング | 残る(設計・要件) | **残る**(レビュー・テスト・障害対応) | AI が省力化 |
| マーケティング分析 | 残る(問い・仮説) | **残る**(市場変化の解釈・戦略決定) | AI が省力化 |
:::

### AI による執行が完結しない三つの理由

ここを慎重に扱いたい。「AI が執行を全部やる」と単純化すると、現場
の実態を見誤る。

**1. AI 導入で、要求される精度水準が上がる**

AI が大量に処理するからこそ、わずかな誤り(ハルシネーション、文脈
誤読、古い規範の適用)が **大量に複製・展開される**。1 件の人間ミス
が 1 件で済んだ時代と比べて、AI のミスは **指数関数的に拡散** する。
結果として、検証フェーズでの精度要求は **AI 導入前より上がる**。
「AI の出力を見て採用する」判断は、執行を自動化したからこそ、より
高度になる。

**2. 環境変化への対応は、AI では完結しない**

AI の訓練データは **特定時点の知識** である。法律・規制・税制・業界
慣行・市場条件・技術スタックは絶え間なく変化する。これらの変化が
起きたとき、AI の出力は **古い前提のまま** 出てくる。

- 新しい税制が施行されれば、AI の経理出力を人間が **見直す**
- 新しい規制が施行されれば、AI の法務チェックを人間が **再構成する**
- 新しい技術が出れば、AI のコード提案を人間が **適応させる**
- 顧客の業界事情・社内文脈が変われば、AI の文書を人間が **書き直す**

これらは「最終判断」の一部ではあるが、単なる承認・却下ではなく、
**AI の出力そのものを修正する** という能動的な作業である。

**3. エッジケースで AI は静かに失敗する**

定型的な業務の 80〜90% は AI で執行できる。しかし残りの 10〜20%
の **エッジケース・想定外・複雑な交差ケース** で AI は失敗する。
**失敗が静か** であることが厄介で、検出と修正には熟練した人間の
経験が要る。AI 出力を「いつも正しい」と扱うと、たまの致命的な誤り
を見逃す。

> したがって、AI 化で確実に消えるのは「**指示通りに、変化のない
> 環境で、単純な執行を繰り返す仕事**」に限定される。
> 「**検証する**」「**環境変化に対応する**」「**エッジケースを
> 修正する**」仕事は、むしろ **AI 化と並行して増える**。

### 規模感は慎重に

つまり、企業内では **総合職 vs 一般職、社員 vs 派遣・BPO・SIer** の
構造として、指示・検証・判断を担う側は残り、純粋な執行を請け負う側
が縮む。ただしこの「縮む」の幅は、

- 業界の環境変化の速さ(変化が速いほど人間の修正作業が多い)
- AI 出力の検証コスト(検証が困難な領域ほど人間が必要)
- エッジケースの頻度

によって大きく違う。**「数百万人が一斉に不要になる」と単純化して
語るべきではない**。

:::highlight
**起きるであろう構造変化の幅(慎重な見積もり)：**
日本の就業者約 6,700 万人のうち、事務従事者は約 1,300 万人。
このうち、**他人の指示で純粋執行のみを行う部分** に該当する労働は、
業務時間で見ると **半分前後** と推定される。
そのうち AI が高い精度で執行できる範囲は、業界・職種で大きく
ブレる。**数百万人規模の労働量が AI に置き換わる可能性は高い**
が、それが「人員ゼロ」を意味するわけではない。一人あたりの
業務構成が変わる(検証・修正・適応の比率が上がる)ことが先に起きる。
:::

### 「指示・判断側」に回れる人の数には上限がある

ここまで「執行が AI に代替されても、指示・要件設計・検証・修正・
適応・最終判断は人間に残る」と書いてきた。これは事実だが、もう
一つ重要な事実がある。**その仕事を実際に担える人の数には上限が
ある**。理由は二つ ──「**できない**」と「**やりたくない**」。

#### できない ── 能力的な上限

執行を巻き取る側に回るには、

- 業務領域の **専門知識**(税制・法務・業界慣行・技術スタック・市場)
- AI 出力の異常を見抜く **批判的読解力**
- 環境変化を察知し、既存出力を更新する **判断力**
- エッジケースで「AI が静かに失敗している」ことに気づく **経験**
- 不確実な状況で意思決定する **覚悟と責任**

が要る。これらは数年〜十数年単位で蓄積されるものであり、**全員が
獲得できるものではない**。

#### やりたくない ── 選好的な上限

加えて重要なのは、**これらの素養を持っていても、やりたくない人が
相当数いる** という事実である。

- 常時オンの責任(問題が起きれば自分のところに戻ってくる)
- 不確実性下の意思決定(間違えれば自分の評価に響く)
- AI の誤りを最終的に被るリスク(承認した自分の責任になる)
- 多面的な認知負荷(複数の領域を同時に俯瞰する疲労)
- 純粋執行のシンプルさからの離脱(「言われたことをやる」のラクさを失う)

これらを引き受けたくない人は、能力があっても引き受けない。これは
個人の **選好** の問題であり、社会的に強制すべきものではない。
かつて「総合職へ昇進できるのに、一般職を選ぶ人がいる」と言われた
構造は、ホワイトカラー全体に拡大して現れる。

「やりたい人」と「やりたくない人」の比率は、業界・年齢・家庭環境・
報酬体系によって大きく違う。現代日本では、若年層を中心に
「**ワークライフバランスを優先したい、責任を負いたくない**」傾向
は強まっている。

#### 結果 ── 指示・判断側の供給は需要に追いつかない

「できない × やりたくない」の二重制約で、指示・判断側に回れる人の
**実質的な供給** は、AI 化で生じる需要に追いつかない可能性が高い。
AI 普及で「純粋執行の労働需要」が大幅に減ったとき、そこから外れた
人の **全員が指示・判断側に再就職できるわけではない** ── かつ、
**全員がそれを望むわけでもない**。

実際に起きるのは、おおよそ三層化:

:::compare
| 層 | 役割 | 起きること |
| --- | --- | --- |
| **熟練者層** | 指示・要件設計・最終判断 | 1 人で 20 人分を回せる、報酬は上がる |
| **中堅の検証・適応担当** | AI 出力の検証・修正・環境適応 | 新しく生まれる中間層、ただし規模は限定的 |
| **純粋執行のみを担っていた層** | 受託・派遣・BPO | **仕事の総量が大幅に減る、移行先が必要** |
:::

これは「全員が高度で多面的な仕事に移れる」という美談ではない。
**労働市場の二極化** が起き、能力的・年齢的・地理的な理由で新しい
役割に移れない人が **相当数生まれる**。本シリーズの構造分析として、
ここを楽観的に書くべきではない。

### 警告 ──「AI エージェントに自律的にやらそう」という危険な逃避

ここで、現在の業界で広く起きていることを警告として書いておく。

「指示・判断側に回れる人が限られる」── この事実に直面したとき、
業界の反応は決まっている。

> 「**じゃあ、検証も判断も AI エージェントに自律でやらせよう**」

GitHub Copilot Workspace、Devin、ChatGPT Operator、Claude
Computer Use、自律 AI エージェント、マルチエージェントシステム
── 2024〜2026 年に大量にリリースされた製品はすべて、この発想に
立っている。**人間がボトルネックなら、人間を抜こう** という発想だ。

**これは構造的に失敗する。** 理由は六つある。

**1. ハルシネーションが互いを補強する**

AI が AI の出力を検証する構造では、誤りはキャンセルされない。
**同じ訓練データから来る同じ偏りが、検証側にも現れる**。一人称
で「自信をもってウソを書く」癖は、二人称(検証エージェント)でも
同じ。結果として、自律ループは **誤りを修正するのではなく、
互いに正当化する**。

**2. 環境変化に盲目になる**

訓練データは静的だ。検証エージェントも適応エージェントも、同じ
過去のデータから出てくる。**新しい税制・規制・技術・市場条件を
「見たことがない」のは、執行側も検証側も同じ**。人間の介入なしに、
外部世界の変化を取り込めない。前節で「環境変化への対応は AI では
完結しない」と書いたが、検証側を AI にしてもそこは解けない。

**3. エッジケースに弱い側が、検証も担う**

エッジケースで AI は静かに失敗する(前節で論じた)。同じ AI に
検証を任せれば、**その失敗を見逃すのも同じ AI** である。エッジ
ケースに気づくのに必要なのは「これは普通じゃない」と感じる人間
の経験であり、訓練データのパターン認識ではない。

**4. 責任の所在が消える**

何か致命的な誤りが起きたとき、誰が責任を取るのか。指示者か、
検証エージェントか、執行エージェントか、開発元か。**自律ループ
のなかで責任は霧散する**。法的・倫理的・運用的に深刻な問題で
あり、社会的に持続できない。

**5. コストが指数関数的に膨れる**

AI 呼び出しはタダではない。エージェントが他のエージェントを呼ぶ
構造は、1 タスクあたりの API コール数を **100 倍・1,000 倍** に
膨れさせる。経済合理性の観点から、自律ループは多くの場合赤字に
なる ── 本シリーズ第6章で論じた **AGI 経済不採算性** と同じ構造
である。

**6. 攻撃される(プロンプトインジェクションと供給網汚染)**

自律エージェントは、攻撃者にとって極めて魅力的な攻撃対象だ。
プロンプトインジェクションでエージェントを乗っ取り、認証された
権限で大規模に被害を出せる。これは本シリーズ
[第5章「Mythosが来た」](/insights/mythos/) で論じた、Claude
Mythos 出現後の攻撃経済の崩壊と直接接続する。Anthropic 自身の
ガイダンス(`/ai-native-ways/ai-delegation/`)も「**自律モードで
エージェントを走らせるな**」と明確に警告している。

> 自律 AI エージェントへの逃避は、人間のボトルネックを解消する
> どころか、**検証・責任・コスト・セキュリティのすべてを同時に
> 悪化させる**。
> **これは「人手不足を解決する技術」ではない。「人手不足を解決
> したかのように見せる、別の問題の集合」である**。

### だから、土地ベースの仕事への移行が必要になる

この二極化を放置すれば、

- 仕事を失った層の所得が消失する
- UBI 議論への政治的圧力が強まる(しかし第6章で示したとおり、
  UBI は構造的な解決にならない)
- 社会の不安定化(失業・分断・対立)

が同時進行する。

しかし、ここで重要なのは、**社会全体としては「物理的な仕事」が
急速に増える** という事実だ。

- バイオ素材の製造には永久に人手が必要(第2章)
- 土壌の再生・微生物の管理には現場の労働が必要(リジェネラティブ農業)
- 食料生産・林業・地域インフラの維持

これらは AI で代替できない。**AI 化で執行から解放された人々の
受け皿** として、社会的に必要であり、かつ実在する。

ただし、移行は自動では起きない:

- 全員がデスクワーカー → 農業・林業に移れるわけではない(年齢・健康・地理)
- 移行には **数年単位の時間** がかかる(技能習得・移住・農地確保)
- その間の生活保障、職業訓練、農地・林地の再配分などの **政策的
  下支え** が要る(後段の「人口の大移動」節で詳述)

> AI 化の構造変化は、能力ある一部の人を 20 倍生産的にする。
> しかし、そうなれる人は限られる ── これが社会全体の問題になる。
> **残り 19 人は、どこで何をするか**。
> それが、本章の後段(「人口の大移動」)で論じる、土地ベースの
> 仕事への人口再配置という問いの中身である。

これが UBI(ベーシックインカム)の議論に繋がる。
しかし第6章で示したように、UBI は間違いだ。
バイオ素材の製造、土壌の再生、生態系の回復 ──
物理的な仕事は永久に人手が必要だ。
受託型デスクワーカーは純粋執行から解放されるが、**仕事はなく
ならない。変わるだけだ ── 変われる人は変わる、変われない人には
別の道(土地ベースの仕事)が必要になる。**

## 大都市から土地のある場所へ——人口の大移動

軍需産業の世代交代、IT 産業の三段階世代交代、そして受託型デスク
ワークの消滅 ── これらは「衰退」ではなく **構造変化** である。
しかし構造変化は、人と場所の関係を書き換える。

- 新しい軍需産業(ドローン+AI、AI 防衛 SaaS)は、巨大プラットフォーム
  のような **大規模製造拠点を必要としない**。少人数のチームと
  地方分散型の製造で十分機能する
- AI ネイティブ企業は、リモートワークと小さな本社で動く。
  Anthropic、Cursor、Vercel ── 数百人〜数千人規模で、東京一極や
  シリコンバレー一極の集中は構造的に不要
- OSS AI が広がれば、AI 自体が **どこからでもアクセスできるインフラ**
  になり、**特定都市に集まる経済的合理性がさらに弱まる**
- 受託型デスクワーク(SIer・BPO・派遣)の数百万人規模の雇用は、
  指示と判断の側に再吸収されるか、別の業種に移るしかない

新しい産業に乗り換えた人々も、乗り換えられなかった人々も、
**「どこに住むか」という問いに、もう一度向き合うことになる**。

第2章で見たように、化石資源の後にはバイオ素材が来る。
第3章で見たように、化学肥料なしの食料生産が必要になる。
第6章で見たように、バイオ素材の製造には永久に人手が必要だ。

**仕事はある。しかし、その仕事は大都市ではできない。**

:::highlight
**バイオ素材・食料生産に必要なもの：**
木材（建材・繊維・化学原料） → **林地**の管理と伐採・植林が必要
植物原料の栽培 → **農地**が必要
リジェネラティブ農業 → **広大な農地**と放牧地が必要
野菜の栽培実践 → **圃場**が必要（実践者が試行錯誤する場）
微生物・菌糸体の培養 → **土地とスペース**が必要
バイオマスの収集 → 農村部・山林での物理的労働
**東京のオフィスビルの中では、何一つできない。**
:::

:::chain
**人口移動の構造：**
旧来軍需産業の縮小 → 都市部の防衛関連雇用が再配置される(新興ドローン+AI 企業は地方分散しやすい)
既存 IT 企業の縮小 + AI ネイティブ企業の台頭 → 都市オフィスへの集中理由が薄れる
デスクワーカーの激減 → 都市部に人を集める理由がなくなる
→ 同時にバイオ素材・食料生産の人手が足りない
→ これらの仕事は土地がないとできない
→ **人口が大都市から地方へ移動する**
:::

考えてみれば、日本の地方の町は既にこの規模で存在している。
商店がある。学校がある。診療所がある。農地と林地に囲まれている。
**大きく変わるのは地方ではない。東京だ。**

:::compare
| | 現在 | 転換後 |
| --- | --- | --- |
| 東京圏 | 3,600万人がオフィスに通勤 | 集中する理由が消える |
| 地方の町 | 過疎化が進行 | 人が戻ってくる（受け皿は既にある） |
| 主な雇用 | デスクワーク、IT、サービス業 | バイオ素材製造、食料生産、林業、土壌再生 |
| 必要な資源 | オフィス、データセンター、電力 | 農地、林地、水、日照、土壌 |
:::

東京一極集中は、化石資源とデスクワークの時代の産物だ。
デスクワークが消え、仕事が土地ベースに変われば、東京に3,600万人が住む理由がなくなる。

地方の暮らし自体はそれほど変わらない。
町があり、サービスがあり、農地と林地がある——その構造はそのまま受け皿になる。
**変わるのは、東京一極集中が解消されることだ。**

ただし、受け皿があるからといって、人が自然に移動するわけではない。
**政策的な後押しが必要だ。**

:::highlight
**必要な政策：**
農地・林地の取得支援 → 都市部からの移住者が土地を持てる仕組み
職業訓練 → デスクワーカーがバイオ素材製造・農業・林業を学ぶ場
住居の初期支援 → 空き家の活用、移住者向け住居の整備
地方の町のサービス維持 → 診療所・学校・商店を維持する財政措置
実践圃場の整備 → 野菜の栽培実践を地方で行う場。研究者ではなく実践者が必要。分析はAIが行う
:::

そしてもう一つ、最も重要な政策がある。
**自由貿易の見直しだ。**

今までの自由貿易は、ある前提の上に成り立っていた。

:::chain
**自由貿易の前提：**
安い石油 → 安い輸送コスト → 地球の裏側からモノを運んでも採算が合う
→ 人件費の安い国で作り、先進国で消費する
→ 国内の農業・林業は「非効率」として切り捨てられる
→ **完全な自由貿易が正しいとされた**
:::

この前提自体が、化石資源に依存していた。

国内でバイオ素材・食料・木材の生産体制を作っても、
完全な自由貿易のままでは、海外からの安い輸入品に潰される。
土地ベースの仕事で暮らす人々の生活が成り立たない。

:::chain
**自由貿易を維持した場合：**
国内で農業・林業・バイオ素材の生産を始める
→ 海外から安い農産物・木材が輸入される
→ 価格競争で国内生産者が生活できなくなる
→ 人が地方に留まる理由がなくなる
→ **産業構造の転換自体が失敗する**
:::

化石資源が枯渇すれば、いずれ輸送コストは上がり、
遠くから安いものを運ぶモデル自体が成り立たなくなる。
しかし、その前に国内の生産基盤が潰れてしまっては意味がない。
**自由貿易の前提が崩れるなら、貿易政策も変えなければならない。**

産業構造の転換は避けられない。
しかし、転換が起きてから慌てるのではなく、
**今から地方の受け入れ体制と貿易政策を整えておく**ことが、政策の仕事だ。

:::highlight
地方の過疎化は「解決すべき問題」ではなく、
**産業構造の転換が解決する結果**になる。
ただし、それを円滑に進めるための政策は必要だ。
:::

## 現在の医療・年金は、この社会に合わない

人口が分散し、仕事が土地ベースに変わる。
この構造転換に対して、現在の医療・年金制度は根本的にミスマッチだ。

:::chain
**現在の制度の前提：**
都市に人口が集中している → 大規模病院・透析施設を都市部に集約
サラリーマンが大半 → 給与から天引きで社会保険料を徴収
65歳で定年退職 → 年金で余生を過ごす
デスクワークで体を壊す → 医療で治す

**転換後の社会：**
人口が地方に分散 → 都市集約型の医療インフラが届かない
土地ベースの仕事 → 給与天引きモデルが合わない
身体を使う仕事を長く続ける → 「65歳で引退」の前提が消える
自然な食と運動 → そもそも病気になりにくい
:::

現在の制度は、化石資源と都市集中の時代に設計された。
その時代が終わるのに、制度だけが残っている。

## 透析医療——化石資源で命を繋ぐ構造

日本の透析患者数は2024年末で337,414人。
国民の約367人に1人が透析を受けている。
患者一人あたり年間約480万円。マクロ総額は**年間約1.6兆円**。

:::chain
**透析医療の化石資源依存：**

ダイアライザー（人工腎臓） → ポリスルホン膜＋ポリカーボネート外筒 → **石油化学製品**
血液回路・穿刺針 → ポリ塩化ビニル（PVC）チューブ → **石油化学製品**
透析液バッグ → プラスチック素材 → **石油化学製品**
超純水の精製 → RO装置の常時稼働 → **大量の電力**
透析液の加温 → 37度まで加熱 → **大量の電力**
14万台のコンソール → 空調・駆動 → **大量の電力**
:::

透析治療の63.3%が血液透析濾過（HDF）——従来型より大量の超純水と電力を消費する。
在宅透析はわずか3.4%。世界最低水準だ。
全国4,512施設に148,339台のコンソール。
この巨大インフラは、化石資源が止まれば機能しない。

:::highlight
**2026年イラン戦争の直撃：**
オイルショックにより電力・材料・物流費が同時に高騰。
透析施設のコスト構造が崩壊する。
患者一人あたりの年間費用は480万円から600万円超へ。
患者数は減少しているのに（337,414人→推計325,380人）、
総医療費は1.6兆円から**2兆円規模へ上方爆発**する。
人口動態の恩恵が、インフレに完全に食われる。
:::

2026年度の診療報酬改定で、光熱水費の補填はわずか**+0.09%**。
オイルショックの現実とは絶望的に乖離している。
中小の透析クリニックは経営不能に陥り、
「透析難民」が発生するリスクが現実化する。

## 社会保険料30%——現役世代の限界

透析は氷山の一角だ。
問題は、日本の社会保障制度全体の構造にある。

:::chain
**現役世代が背負う負担：**

健康保険料 → 約10%
厚生年金保険料 → 約18.3%
介護保険料 → 約1.8%
雇用保険料 → 約0.9%

**合計 → 約31%**（労使折半前の実質負担）
:::

給与の3割以上が、働く前に消える。
しかも、この比率は毎年上がり続けている。
少子高齢化が進めば、40%、50%になる計算だ。

:::highlight
**構造的に成立しない数学：**
1970年：現役世代8.5人で高齢者1人を支えていた
2020年：現役世代2.1人で高齢者1人を支えている
2040年：現役世代1.5人で高齢者1人を支える予測
1.5人で1人を支える社会保障は、どんな制度設計でも破綻する。
:::

## 年金制度——デスクワーク時代の遺物

現在の社会モデルは「65歳まで働く→年金で余生を過ごす」だ。

この制度が設計された時、前提は二つあった。
「余生は短い」と「人口は増え続ける」だ。

1960年代、日本人の平均寿命は65〜67歳。年金支給は55歳から。
余生は10年程度の想定だった。

いま、平均寿命は84歳。年金支給開始は65歳。
余生は20年。しかも、その20年間に医療費が膨大にかかる。

:::chain
**年金の構造的矛盾：**

長寿化 → 年金支給期間が20年以上に
少子化 → 支える現役世代が減少
医療の高度化 → 高齢者の医療費が増大
社会保険料 → 現役世代の30%超

働く人が減り、遊ぶ人が増える → **数学的に破綻**
:::

しかし、問題はもっと根本的だ。
**「65歳で引退する」という概念自体が、デスクワーク時代の産物ではないか。**

:::highlight
**デスクワーク社会の年金モデル：**
オフィスで座って働く → 運動不足＋加工食品 → 体を壊す
体を壊す → 医療に依存する → 医療費が膨張する
65歳で体が動かなくなる → 年金で養う → 現役世代が負担する
これが「先進国」の社会モデルだった。
:::

転換後の社会では、この前提が崩れる。

:::chain
**土地ベースの仕事がもたらす構造変化：**
身体を使う仕事 → 運動が生活に組み込まれる
自分で育てた食物 → 加工食品への依存が減る
自然な食と運動 → 生活習慣病のリスクが下がる
→ 医療への依存が減る
→ 65歳で「引退」する必要がない
→ 年金への依存が減る
:::

「病気を治す」のではなく「病気にならない」。
「年金で生活する」のではなく「年金が不要な暮らし」。
これは福岡正信の「何もしない農業」と同じ構造の逆転だ。

自然から離れるほど、生きるコストは上がる。
自然に近づけば、制度に頼る必要が減る。

## アメリカ農業の構造的破綻

「日本の農業は非効率だ、アメリカのように大規模化すべきだ」
——この主張は、アメリカ農業の構造を全く理解していない。

:::chain
**アメリカ農業の実態：**

大規模モノカルチャー → 同じ作物を広大な土地に → 土壌微生物の多様性が消滅
化学肥料への完全依存 → 窒素・リン・カリウムを外部から投入 → 土壌は「培地」に退化
大量の農薬 → 除草剤（グリホサート等）→ 土壌微生物をさらに殺す
大型機械 → 土壌を圧縮 → 水と空気の循環を破壊
灌漑への依存 → オガララ帯水層の枯渇 → 回復に数千年
:::

:::highlight
**表土の喪失——取り返しのつかない損失：**
アメリカの農地は、過去150年で表土の約50%を失った。
表土1インチ（2.5cm）の形成には500〜1,000年かかる。
大規模農業は、数千年かけて蓄積された土壌資本を、
数十年で使い果たしている。
これは採掘であって、農業ではない。
:::

:::highlight
**オガララ帯水層——枯渇する地下水：**
アメリカ中西部の穀倉地帯を支える巨大地下水脈。
年間の汲み上げ量が、自然補給量の数倍〜数十倍。
カンザス州やテキサス州では、すでに水位が30m以上低下。
完全な回復には数千年〜数万年かかる。
アメリカの「食料安全保障」は、枯渇する地下水の上に建っている。
:::

:::compare
| 項目 | アメリカ大規模農業 | 自然農法 |
| --- | --- | --- |
| 土壌 | 消耗品（採掘して使う） | 生きたシステム（育てる） |
| 肥料 | 外部依存（化石資源由来） | 不要（微生物が供給） |
| 水 | 地下水を採掘（枯渇する） | 雨水＋土壌の保水力 |
| 農薬 | 必須（モノカルチャーの宿命） | 不要（生態系バランス） |
| エネルギー | 大量消費（機械＋輸送） | 最小限（人力＋自然力） |
| 期間 | 持続不可能（数十年で土壌崩壊） | 持続可能（数千年の実績） |
| 危機耐性 | 極めて脆弱（サプライチェーン依存） | レジリエント（自立型） |
:::

アメリカ農業は「効率的」に見えるが、
それは土壌と地下水という資本を食い潰しているからだ。
銀行口座の残高を使い果たしながら「収入が多い」と言っているに等しい。

## 日本の本当の資源——水

「日本には資源がない」

これは日本で最も広く信じられている誤解だ。

日本には**水**がある。これが最大の天然資源だ。

:::chain
**水が豊かな国の構造的強み：**

年間降水量 → 日本は約1,700mm（世界平均の約2倍）
水源 → 山岳地帯が国土の73%。天然の貯水・浄水システム
河川密度 → 極めて高い。どこでも水にアクセスできる
地下水 → 豊富で清浄
温暖湿潤気候 → 植物の成長に最適
:::

:::highlight
**水がなぜ最大の資源か：**
石油は代替できる（太陽光、風力、核融合）。
水は代替できない。
農業には水が必要。
微生物には水が必要。
植物には水が必要。
人間には水が必要。
アメリカのオガララ帯水層は枯渇している。
中東は慢性的な水不足。
中国北部は深刻な水危機。
日本は、毎年空から1,700mmの水が降ってくる。
これを「資源がない」と言うのは、
石油とレアメタルしか「資源」と認識していないからだ。
:::

水があれば、植物が育つ。
植物が育てば、微生物が育つ。
微生物が育てば、土壌が再生する。
土壌が再生すれば、食料が生まれる。
食料があれば、人間は生きられる。

:::chain
**日本の本当の資源循環：**

豊富な雨水 → 多様な植物 → 土壌微生物 → 肥沃な土壌 → 食料
↑ ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ←（循環）← ↓
:::

石油は掘れば減る。水は循環する。
これが「資源」の本質的な違いだ。

## 福岡正信の思想——生きるためにどうするか

全ての構造分析は、一つの問いに収束する。

**生きるためにどうするか。**

福岡正信の自然農法は、この問いへの答えだ。

:::chain
**福岡正信の四つの原則の本質：**

無耕起 → 土壌の構造を壊さない → 微生物が生きる
無肥料 → 外部依存をゼロにする → サプライチェーンに縛られない
無農薬 → 生態系を殺さない → 自然の防御力を活かす
無除草 → 光合成面積を最大化 → Light Farmingの原理
:::

そしてChristine Jones博士のLight Farmingが、
この思想の科学的根拠を与えた。

:::chain
**Light Farming——福岡正信の科学的証明：**

太陽光 → 光合成 → 植物が糖を作る
糖 → 根から土壌へ滲出（Liquid Carbon）
Liquid Carbon → 土壌微生物のエネルギー源
土壌微生物 → 植物に必要なミネラルを可溶化して供給
植物 → さらに光合成 →（正のフィードバックループ）

**雑草も光合成する。草を残せば、Liquid Carbonの供給源が増える。**
福岡正信の「無除草」は、Light Farmingの最大化戦略だった。
:::
