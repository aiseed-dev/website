# 伝統野菜・料理辞典 — 計画書 v3（現行方針）

作成日: 2026-06-13 ／ 正本: 本書 + `../README.md`

> v2（2026-01）からの転換を反映した現行版。要点は一つ：
> **AI は「たたき台」を機械の速さで出す。確定するのは人間。** 全部を AI で
> 構造化 DB に積む発想（v2）から、**人間が確定した原稿を正本とする静的サイト**へ。
> これは本サイトの構造分析（[生得性と観察 — AIと農業の同型](/insights/nativism-observation/)）
> ＝「構造は予め存在する。設計は足すのではなく読む」と同じ思想である。

## 1. 何を作るか

イタリア・日本の伝統野菜と伝統料理の辞典サイト。まずは**イタリア野菜の図鑑**から。
各作物を **概要・歴史・栽培・料理** の4分類で記述する。

## 2. データパイプライン（正本）

```
① data/deep_research/イタリア野菜/   Gemini 調査報告書（一次資料）
      │ AIが要約
② data/deep_research/italian/<作物>/  AI要約（たたき台）
      │ ★人間がレビュー・修正
③ web/italian/                        正本（人間確定稿）
      │ web/build.py
④ web/site/italian/                   静的HTML（.gitignore）
      │ 相対シンボリックリンク
   html/vegitage/italian               本体の公開ディレクトリ
      │ tools/cloudflare_pages_deploy.py（リンクを辿る）
   Cloudflare Pages（/vegitage/italian/…）
```

- **たたき台②の生成**：`src/agents/research_min.py`（Claude Agent SDK の one-liner パターン・
  WebSearch・出典必須）。任意。手で書いてもよい。
- **正本③は人間**：`web/italian/` で確認・修正して確定する。自動公開しない。
- **検証**：辞典には「現行システム」のような正解器が無い。事実（DOP/IGP・学名・産地・統計）は
  **出典＋業務知識を持つ人間**で確かめる。

## 3. ファイル構成（1作物）

```
web/italian/<作物>.md            概要（YAML フロントマター + 短い概要文）= トップ
web/italian/history/<作物>.md     歴史
web/italian/cultivation/<作物>.md 栽培
web/italian/cuisine/<作物>.md     料理
```

### 概要フロントマター（スキーマ）

```yaml
---
id: IT-VEG-ART-001
name_ja: アーティチョーク
name_it: Carciofo
name_en: Artichoke
aliases: [チョウセンアザミ]      # 別名・地方名・別表記
family: キク科                  # 科（植物学的事実・検索/参照）
family_latin: Asteraceae
botanical: Cynara cardunculus var. scolymus   # 種の学名
index_group: キク科             # 目次「科」タブのグループ名（family と別管理・自由命名）
type: [花菜類]                  # 食用部位。複数可。概要先頭に表示・目次「type」タブ
item: アーティチョーク
certification: [DOP, IGP]
regions: [Lazio, Sardegna]
season: [春]
uses: [加熱, 保存]
hero_image: images/hero.jpg
---
```

## 4. 目次（index）

ビルド時に**静的生成**し、2つのタブで切り替える（JSON も client-side 検索も使わない）：

- **「科」タブ** → `index_group` ごとにグループ表示
- **「種類」タブ** → `type` ごと（複数所属可）

free-text 検索は当面**保留**（タブ閲覧で探索要件を満たす。必要になれば小さな JSON+JS を足す）。

## 5. AI の役割と道具

| 道具 | 役割 | 状態 |
|---|---|---|
| `src/agents/research_min.py` | たたき台②の生成（最小・出典付き） | 現役 |
| `web/build.py` | 正本ビルダー（③→④ 静的HTML） | 現役 |
| `src/tools/gen_icons.py` | アイコン生成（Gemini）→ web/static/icons | 補助 |
| `src/tools/extract_varieties*.py` | 品種抽出（検索用） | 保留 |
| `src/agents/{research_agent,gemini_agent,batch_runner,pdf_converter}` | 大規模調査・JSON DB 生産 | 保留 |
| `src/schemas/`・`src/validators/` | 構造化JSON DB のスキーマ/検証 | 保留 |

## 6. 技術と原則

- **静的サイト**（HTML）＋ **Cloudflare Pages**。サーバー無し・バックエンド無し。
- **データは手元**（`web/italian` の Markdown）。ベンダーロックインなし。対象は公開知識（機微データなし）。
- 公開は本体サイトに相乗り：`web/site/italian` を `html/vegitage/italian` にシンボリックリンク。
- ツールは小さく：`markdown` + `pyyaml` で十分。依存は `vegitage-data/.venv` に隔離。

## 7. 保留（将来）

v2 の壮大な構想のうち、いまは止めて将来に回すもの：

- **構造化JSON DB**（850品種/1300レシピ、`data/{vegetables,recipes}`）と**野菜↔料理の双方向リンク**。
- **日本の野菜・料理**（`master_lists` はヘッダのみで未着手）。
- **free-text 検索**（品種データ `extracted_varieties` の活用を含む）。

これらは「正本（人間確定の静的サイト）」が育ってから、必要に応じて積み増す。
**先に全部 DB に積まない** —— それが v2 からの最大の学びである。
