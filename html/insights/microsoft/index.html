Microsoftの破産

Six Taxesが剥がれる時、何が起きるか。

2025.04.03

構造分析シリーズ 07

Microsoftの6つの税

Microsoftのビジネスモデルは、バンドルだ。 Windows、Office、Azure、Teams、Dynamics、LinkedIn—— これらを組み合わせて売ることで、ユーザーは気づかないうちに 複数の「税」を払わされている。

この6つの税を構造的に分解する。 1 OS税 — Windows PCメーカーがWindowsライセンスを組み込み、ユーザーは購入時に自動的に支払う。選択の余地なく課される税。Linux、ChromeOSという代替は存在するが、「互換性」という鍵でロックインされている。 2 生産性税 — Microsoft 365 Word、Excel、PowerPoint、Outlook。「全員が使っている」という事実がロックイン。Google Workspaceという完全な代替が存在し、多くの用途でより優れているにもかかわらず、慣性で支払い続けている税。 3 コミュニケーション税 — Teams Microsoft 365にバンドルされ、「無料で使える」と思わせて浸透させた。EUがバンドル販売を独禁法違反と認定。Slack、Zoom、Google Meetという代替がある。 4 クラウド税 — Azure Windowsとの「シームレスな統合」を売りにするが、実態はベンダーロックイン。AWS、GCPという同等以上の代替が存在。Azureの技術的優位性はほぼない。 5 開発者税 — GitHub + VS Code GitHubを買収し、VS Codeを無料配布し、開発者のワークフロー全体を囲い込んだ。Copilotはこのエコシステムに組み込まれている。 6 AI税 — Copilot 既存の全製品にCopilotを組み込み、追加料金を課す。Copilot for Microsoft 365は月額30ドル/ユーザー。既にMicrosoft 365の料金を払っている上に、さらに課される税。

Copilotの言語的思考 vs. Claudeの構造的思考

Six taxesの中で最も脆弱なのがAI税だ。 なぜなら、Copilotは言語的思考の道具であり、 構造的思考の道具ではないからだ。

特性 Copilot（言語的思考） Claude（構造的思考）
--- --- ---
設計思想 既存の製品群を「AIで強化」 思考プロセスそのものを支援
得意なこと 文書の要約、メールの下書き、表の整形 構造の分析、因果関係の追跡、分野横断的な思考
ユーザーに求めるもの 「何をしてほしいか」を言語で指示 「何を調べるべきか」を構造で指示
出力の性質 テンプレート的、予測可能 構造的、文脈依存的
Claude Code相当 なし（バンドル内の機能として存在） Claude Code — 構造的思考者のための統合環境

Copilotは「既存のMicrosoft製品をAIで少し便利にするもの」だ。 Wordの中で文章を書かせる。Excelの中で数式を提案させる。 Teams会議を要約させる。

これらはすべて言語的なタスクであり、 「肥料の生産ルートを追跡して構造を可視化する」ような 構造的思考は設計の射程に入っていない。

根本的な違い： Copilotは「製品の付加価値」として設計されている。 Claudeは「思考の道具」として設計されている。 製品の付加価値は、製品がなくなれば消える。 思考の道具は、思考者がいる限り価値を持つ。

税が剥がれるメカニズム

Six taxesはバンドルによって相互に強化されている。 しかし、バンドルの強度は「最も弱い税」で決まる。 一つが剥がれると、連鎖的に他の税も剥がれ始める。

崩壊のシナリオ：
AI税が剥がれる（Claude/Claude Codeが優位）
↓
開発者税が剥がれる（Claude CodeがVS Code + Copilotを代替）
↓
生産性税が剥がれる（AIが直接文書を生成、Officeが不要に）
↓
コミュニケーション税が剥がれる（AIがコミュニケーションを再設計）
↓
クラウド税が剥がれる（AIワークロードがAzureに依存しない）
↓
OS税が剥がれる（ブラウザ/AIがOSを抽象化）

この連鎖が始まるトリガーは何か。 構造的思考者がCopilotを使わなくなることだ。

構造的思考者は少数だが、技術選定に対する影響力は大きい。 彼らが「CopilotよりClaude Codeの方が生産性が高い」と判断し、 チームに導入し始めた時、税の剥離が始まる。

アルトマンもナデラもGoogleも見えていないもの

OpenAIのアルトマン、MicrosoftのナデラGoogleのピチャイ—— 彼らはClaudeの本当の強さが見えていない。

なぜか。構造が見えないからだ。

彼らはAIを「言語処理の進化」として見ている。 より大きなモデル、より多くのパラメータ、より速い推論—— これらの指標で競争している。

しかし、Claudeの価値はモデルサイズではない。 構造的思考を支援するアーキテクチャにある。 Claude Codeは、構造的思考者がコードベース全体を見渡し、 関連する構造を横断的に追跡するための道具として設計されている。

見えていないもの： アルトマンはベンチマークを見ている。 ナデラはバンドルの売上を見ている。 ピチャイは検索シェアを見ている。 誰も「構造的思考の増幅」を見ていない。 見えないものとは競争できない。

なぜ「破産」と言えるのか

「破産」は挑発的な表現だが、構造的には正確だ。

Microsoftの収益構造は、6つの税の重ね合わせで成り立っている。 一つ一つの税は、より優れた代替手段が存在するにもかかわらず、 バンドルと慣性によって維持されている。

AIの進化がバンドルを無意味にする。 なぜなら、AIは「製品群の中の機能」ではなく 「製品群を代替する思考環境」だからだ。

Wordの中のCopilotで文書を書くのではなく、 Claudeに直接文書を生成させる。 Excelの中のCopilotでデータを分析するのではなく、 Claude Codeに直接分析コードを書かせる。

製品がレイヤーとして不要になった時、 その上に課されていた税も消える。 6つの税すべてが同時に消える可能性がある。 それが「破産」の構造的意味だ。

Microsoftは6つの税を同時に課すことで巨大化した。
6つの税が同時に剥がれることで崩壊する。
バンドルで築いた帝国は、バンドルの解体で終わる。

結論： Microsoftのバンドルモデルは6つの税をユーザーに課している。 Copilotの言語的思考は、Claudeの構造的思考に勝てない。 AI税が剥がれることで、他の5つの税も連鎖的に剥がれる。 構造的思考者がClaude Codeに移行した時、崩壊が始まる。
