# AI時代のデスクワークとシステム設計——Excel・Word 操作をコードにする

## デスクワークの大半は Excel と Word の操作

今のデスクワークを思い出してほしい。

- Excel でデータを貼り付けて、整形して、集計する。
- Word で報告書を書く。体裁を整える。
- PowerPoint でスライドを並べる。
- Outlook でメールを仕分けする。

業務時間の多くが、**バイナリファイルの手作業**に消えている。判断していない。単に操作している。

これを**そのままシステム化**するのが、AI時代のデスクワーク設計だ。

## Office そのものが AI 時代に通らない

Copilot 以前に、**Office（Word / Excel / PowerPoint）そのもの**が AI 時代には構造的に苦しい。

- **バイナリファイル**：中身は AI にも人間にも読みにくい。`git diff` で差分が取れない。
- **履歴が残らない**：「誰が・いつ・なぜ変更したか」は組織の頭の中にしか残らない。
- **手作業が中心**：セルを埋める、書式を整える、レイアウトを直す——判断していない操作が業務時間を食い潰す。
- **年間数百件の脆弱性**：ファイルを開くだけ、プレビューするだけで攻撃が成立するゼロデイが毎年のように出る。
- **密結合**：Windows ＋ Office ＋ Entra ID ＋ Azure ＋ SharePoint が密に結びついていて、1箇所の突破が全体に波及する。

Office は「個人 PC 時代の文書中心モデル」のまま肥大化してきた。AI が読み書きできる形でもないし、現代の開発・監査・共同作業の流儀にも合わない。

**Copilot の自動化は、この根本問題を放置したまま、上に脆弱な AI エージェントを重ねているだけだ。** 土台が AI 時代に合わないのに、便利さで覆ってしまう。Mythos 時代には、この構造は通らない。

## 2026年4月、道具が揃った

2026年4月、画期的な道具が2つ揃った。

- **Claude Mythos**（2026年4月、Anthropic が Glasswing 経由で限定公開）：高度な論理推論、抽象的思考、政策・戦略立案、そしてコーディング。
- **Gemma 4**（2026年4月、Google が Apache 2.0 で公開）：高性能なオープンソースモデルが、**手元のマシンで、外部通信ゼロで動く**。高速処理、定型業務、完全なプライバシー保護。無料。

Claude がロジックをコードにする。Gemma 4 が手元で定型処理をこなす。**この2つで、Excel/Word 操作はコードに置き換えられる。**

## 緊急——Mythos公開までに終わらせる

2026年4月7日、AnthropicはProject Glasswingを発表した。Claude Mythosは12社＋40超のインフラ組織に限定される。**Glasswingの外側の組織は、自分で守るしかない。**

**これは「いずれやればいい」話ではない。締切がある。**

Mythosは永久に封印されない。Glasswingの12社の修正が進めば、一般公開される。その瞬間、**防御側と攻撃側が同時にMythosを使い始める。**

### 最大の問題——Windows / Office は間に合わない

緊急の最大の理由はここだ。多くの組織が依存している Windows と Office は、数十年分のレガシーコードベースを抱え、さらに Copilot が OS と全 Office 製品に統合されている。Mythos でスキャンしても、修正すべき箇所が膨大すぎて公開までに終わらない。

### Copilot の「AI自動化」は Mythos 時代に使えない

Microsoft は Excel/Word の操作負担を **Copilot エージェントによる自動化** で解決しようとしている。しかしこの方向は、Mythos 時代には構造的に成立しない。

:::chain
**Copilot 自動化の構造：**
Office 文書（メール、表計算、契約書）→ Copilot がすべて読む
→ 外部 AI サービスに送信 → 応答が Office に戻る
→ 文書に仕込まれた指示が Copilot を乗っ取れる（プロンプトインジェクション）
→ **攻撃面は文書の数だけ増える。Mythos 公開後は即座に攻撃対象になる。**
:::

- **プロンプトインジェクション**：文書に仕込まれた指示で Copilot が乗っ取られる。
- **データ主権の喪失**：社内文書が外部 AI に送信される。
- **思考の退化**：考える作業まで任せると、人間の判断力が育たない。

Excel/Word 操作を減らすべきなのは正しい。しかし**方法が違う**。Copilot は「便利だが脆弱で検証できない自動化」を増やしている。Mythos が来ればそれが全部入口になる。

### 残された時間

:::chain
**カウントダウン：**
Glasswingの12社の修正 → Mythos一般公開
→ 攻撃側も同じ能力を手に入れる
→ CMS、旧来のDB、Copilot 統合文書——全部スキャンされる
→ Microsoftのレガシーと Copilot 自動化に依存したシステムが最初の標的になる
→ **準備の時間は半年〜1年。公開後では遅い。**
:::

## AIは役割で分ける

能力で比較しない。**役割で分ける。**

### Claude Mythos——考える仕事（外部通信あり）

コーディング、構造分析、戦略立案、執筆。複雑な判断と創造。Claude Code で開発する。渡すのは「考えさせるべき題材」だけ。生データや機密は渡さない。

**Excel 業務を観察する → ロジックを抜き出す → Claude にコード化を依頼する。** これでデスクワークの大半がシステム化される。

### Gemma 4 CLI——処理する仕事（ローカル完結）

要約、集計、変換、定型分析。**外部通信ゼロ。手元のマシンで動く。**

**エージェントとして常駐させない。CLI で呼び、一回で完結する。** 権限を持たせない。「入力→処理→出力」で終わり。

毎日の売上集計、議事録の要約、CSV の整形、メール文面の生成——機密を含む定型処理を、Gemma 4 CLI に一発で任せる。Copilot のように常駐させないから攻撃面を作らない。

### Gemini——調べる仕事

資料収集、決算分析、deep research。Google検索インフラに直結しているので速い。出力は人間が読む。

### 本番環境——AIなし

静的HTML、Python、SQLite。**攻撃面ゼロ。**

本番にAIエージェントを入れない。常時外部通信＋システム権限＋検証できない判断——それは Mythos クラスの攻撃者に入口を提供することだ。

:::chain
**4つの原則：**
考える → Claude Mythos（題材だけ外に出す）
処理する → Gemma 4 CLI（ローカルで完結、常駐させない）
調べる → Gemini（検索統合）
動かす → AIなし（本番は静的・ローカル）
**人間がハブ。AI同士を自動でチェーンしない。**
:::

## デスクワークをシステム化する

### 脱バイナリ——Doc as Code

Excel・Word のバイナリは、AI にも人間にも扱いにくい。

:::compare
| バイナリ中心 | テキスト中心 |
| --- | --- |
| Excel の集計マクロ | Python スクリプト |
| Word の報告書テンプレ | Markdown ＋ 生成スクリプト |
| メールの定型文面 | Gemma 4 CLI による文面生成 |
| 手作業で整形 | 意図 → コード → 成果物（PDF等）を直接生成 |
| 差分が見えない | Git で全履歴 |
:::

文書もコードと同じく Git で管理する。**誰が・いつ・なぜ変更したか**が全部残る。

### システム設計——人間が認めたロジックをコードに

:::chain
**設計の基本：**
Excel/Word でやっている作業を観察 → 判断と定型処理を分ける
→ 定型処理は Claude にコード化させる（Python, SQL, シェル）
→ 100%の再現性で自動実行 → Git に証跡を残す
→ **コード化は Claude がやってくれる。難しくない。**
:::

- **AIエージェントに判断させない。** 動くのは人間が認めたロジックだけ。
- **自動化できないことは、人間が処理する。** その時に AI に助けてもらう。
- **責任は人間が取る。** AI はツール。判断と責任は人間の側にある。

これが Copilot 自動化との本質的な違いだ。Copilot はブラックボックスのエージェントに判断を委ねる。こちらは、人間が認めたロジックを透明なコードとして実行する。

## 実践：5原則

:::highlight
**原則1：AIは役割で分ける。本番には入れない。**
Claude Mythosで考え、Gemma 4 CLIで処理し、Geminiで調べる。本番は静的。
:::

:::highlight
**原則2：脱バイナリ。Doc as Code。**
Markdown、CSV、Python、JSON。全部 Git に入れる。
:::

:::highlight
**原則3：CMSとDBを本番に置かない。**
必要ならローカルで SQLite。SQL インジェクションの対象がなければ攻撃は成立しない。
:::

:::highlight
**原則4：公開サイトは静的にする。**
Linux＋Nginx＋静的ファイルで十分。Gemma 4 は手元のマシンで CLI として動かす。
:::

:::highlight
**原則5：ブラックボックスを排除する。**
コードは AI に任せる。構造と目的を理解するのは人間の仕事。
Copilot のような「見えない自動化」を入れない。
:::

## 自立は孤立ではない

Claude Mythos がコードを書く。Gemma 4 CLI が手元で処理する。Gemini が調べる。**人間がやることは、何を作るか決めて、AI に指示して、結果を判断することだ。**

コードは書かなくていい。構造は理解しておく。

ソフトウェアを AI に任せれば、人間は現実世界の仕事に集中できる。作物を育てる。製品を作る。人と向き合う。**これがデスクワークをシステム化した先の本当の姿だ。**
